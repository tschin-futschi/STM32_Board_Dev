# 开发总结 2026-03-26 22:50

## 变更清单

### 新增文件
- **`protocol.MD`**（由 `protocal.MD` 重命名而来）
  - 开发者手写的串口通讯协议文档，v1.3
  - 定义控制帧格式、数据流帧格式、命令码、错误码、心跳机制、超时重发机制、CRC16-MODBUS 算法规格

### 修改文件

- **`README.md`** — 全面重写
  - 新增硬件规格表（含 UART/I2C/LED 引脚）
  - 新增通信协议概述章节（从 protocol.MD 提炼）
  - 新增开发进度表，反映真实状态
  - 项目结构与实际目录对齐

- **`App/Inc/app_protocol.h`** — 从框架存根升级为完整头文件
  - 新增帧常量宏：`PROTO_SOF1`、`PROTO_SOF2`、`PROTO_STREAM_SOF`、`PROTO_MAX_DATA_LEN`、`PROTO_FRAME_OVERHEAD`、`PROTO_MAX_FRAME_LEN`、`PROTO_CRC_ERR_SEQ`
  - 新增命令码枚举 `Proto_Cmd_t`：`PROTO_CMD_HEARTBEAT`（0x00）、`PROTO_CMD_ERROR_RESP`（0x01）
  - 新增错误码枚举 `Proto_ErrCode_t`：`PROTO_ERR_CRC_FAIL`（0x01）、`PROTO_ERR_UNKNOWN_CMD`（0x02）、`PROTO_ERR_EXEC_FAIL`（0x03）
  - 新增帧结构体 `Proto_Frame_t`（seq、cmd、len、data[255]）
  - 新增命令处理函数指针类型 `Proto_CmdHandler_t`

- **`App/Src/app_protocol.c`** — 从 echo 存根升级为完整实现
  - 新增 8 状态 RX 解析状态机（`ProcessByte()`）：WAIT_SOF1 → SOF2 → SEQ → CMD → LEN → DATA → CRC_H → CRC_L
  - 新增 CRC16-MODBUS 逐字节递增计算（`CRC16_Update()`），CRC 随字节流动态累积，无额外堆栈缓冲区
  - 新增控制帧组包发送（`SendFrame()`），CRC 在组帧时同步计算，大端存放
  - 新增错误回包（`SendErrorResp()`），CRC 失败时帧序号固定填 `0xFF`
  - 新增心跳命令处理（`HandleHeartbeat()`），原样回传 seq，数据段为空
  - `App_Protocol_Init()` 初始化所有状态变量
  - `App_Protocol_Poll()` 从环形缓冲读字节，逐字节喂状态机

- **`CLAUDE.md`**
  - 第 6 节"通信协议"：删除旧的暂定格式（`[SOF 0xAA][LEN][CMD][DATA][CRC8]`）和"允许/禁止"约束表，替换为 protocol.MD 的完整规格摘要
  - 路线图：3.2 状态更新为**完成**
