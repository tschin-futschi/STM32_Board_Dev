# 2026-05-22 16:35 — 修复 I2C2 上电时序（PMIC EnableSequence 前置）

## 一、问题现象

最新固件（base `8a638e4` 起一系列改动后）烧录上电，上位机所有指令全部
timeout：SetPmicVoltage / I2cBusScan / 心跳均无响应，3 次心跳丢失后断开。

```
[motordev.serial] Command timeout: cmd=0X09(SetPmicVoltage) after 3 attempts
[motordev.config] ERRORRESPONSE RX ERRORCODE=0XFF (DISPATCHER TRANSPORT ERROR)
[motordev.serial] Heartbeat lost: 3 missed, disconnecting
```

LED 现象：常灭，未进入主循环。

## 二、定位过程

### 阶段 1：通过 LED 闪烁分组定位卡死 init 阶段

main.c 顶部临时插入 `DiagBlinkN(N)` 助手 + 11 个 checkpoint，每个 BSP/App
初始化成功后 LED 闪 N 次，失败处保持 `while(1)`。

实测：**LED 闪 3 次后常灭** → 卡在 `BSP_I2C2_Init` 失败处。

### 阶段 2：细分 BSP_I2C2_Init 内部失败点

`BSP_I2C2_Init` 唯一 `return ERROR` 出口在 `BSP_I2C2_RecoverBus()` 后
SDA 仍读到 0。新增 `g_i2c2InitDiag` 全局变量，在 return 前记录 SCL 电平
（`1` = SDA 和 SCL 都低，`2` = 只 SDA 低），main.c 失败 halt 改为
`while(1) { DiagBlinkN(g_i2c2InitDiag); }`。

实测：**LED 反复 1 闪** → SDA 与 SCL 同时被拉低，不是 slave 抢 SDA，
而是两条上拉线都没电压。

### 阶段 3：假设 1 — I2C2 上拉接 PMIC 输出 IOVDD

初始化顺序 `I2C1 → I2C2 → I2C3 → PMIC_Init` 中，I2C2 早于 PMIC。
若上拉接 PMIC 的 IOVDD，则 I2C2_Init 时 IOVDD 还没起来 → 两条线都
拉不高。

调整顺序：`I2C1 → I2C3 → PMIC_Init → PMIC_HwenInit → I2C2`。

实测：**LED 还是 6 闪后反复 1 闪** → I2C2 仍失败。假设方向对但还差
一步。

### 阶段 4：根因 — PMIC_Init 仅写电压寄存器，未开 LDO

查 `bsp_pmic.c`：`BSP_PMIC_Init` 只调用 `BSP_PMIC_SetVout()` 预设
LDO1/2/3 电压寄存器，**并不写使能位**。真正打开 LDO 输出的是
`BSP_PMIC_EnableSequence()`（顺序：LDO2 → 10ms → LDO1 → 10ms → LDO3）。
该函数仅在主循环检测到 HWEN 上升沿时被触发，开机阶段从未被调用。

→ IOVDD 一直为 0V，I2C2 上拉（默认电压 280 = 2.80V）没电，
SDA/SCL 均读 0。

## 三、修复方案

main.c 初始化序列中显式调用 `BSP_PMIC_EnableSequence()`，把 LDO 强制
打开，让 IOVDD 上电后再做 I2C2 总线检查。

新顺序：

```
NVIC → SysTick → LED → UART
→ I2C1
→ I2C3
→ PMIC_Init        (预设电压寄存器)
→ PMIC_EnableSequence  (打开 LDO，IOVDD 上电)  ← 新增
→ PMIC_HwenInit
→ I2C2             (此时 SDA/SCL 上拉有电压)
→ aw_isp_init
→ App_Protocol_Init → App_Sample_Init
→ App_Protocol_SendBootStatus(PROTO_BOOT_OK)
→ main loop
```

HWEN 中断逻辑保留：若外部 HWEN 上升沿到来，主循环仍会调用
`BSP_PMIC_EnableSequence()` 重新拉起 PMIC。

## 七、复盘备忘

- `BSP_PMIC_Init` 命名让人误以为它会"启用"PMIC，实际只配置寄存器。命名
  与行为不一致是这次卡了一天的元凶。后续若调整可改名 `BSP_PMIC_PresetVout`
  之类
- `BSP_I2C2_Init` 中的 SDA stuck 检查在 8a638e4 之前是有的，但旧版本
  曾经把 I2C2 init 失败旁路了（`(void)BSP_I2C2_Init();`），自然没暴露
  这个上电时序问题。本次回到严格 halt-on-fail 把问题钉死
- 软件 bit-bang I2C 配置为 `GPIO_PuPd_NOPULL + Open Drain` 完全依赖外部
  上拉；任何让上拉失效的因素（IOVDD 未上电、上拉电阻虚焊等）都会让初始
  化失败。`g_i2c2InitDiag` 的 SCL 电平区分对今后此类问题定位有用
