2硬件层 — GPIO 复用配置

STM32F4 的引脚内部有一个复用选择器（AF Mux），每个引脚可以连接到不同的外设。PA9 和 PA10 通过 AF7 连接到 USART1。

GPIO 复用必须做两步，缺一不可：

/* 步骤①：把引脚模式设为复用（AF），并配置电气特性 */
gpio.GPIO_Pin   = BSP_UART_TX_PIN;     // PA9
gpio.GPIO_Mode  = GPIO_Mode_AF;       // 复用模式，引脚交给外设控制
gpio.GPIO_Speed = GPIO_Speed_50MHz;
gpio.GPIO_OType = GPIO_OType_PP;      // 推挽输出
gpio.GPIO_PuPd  = GPIO_PuPd_UP;       // 上拉，UART 空闲态为高电平
GPIO_Init(BSP_UART_TX_GPIO_PORT, &gpio);

/* 步骤②：把引脚的复用选择器拨到 USART1（AF7） */
GPIO_PinAFConfig(BSP_UART_TX_GPIO_PORT, BSP_UART_TX_PIN_SOURCE, BSP_UART_TX_AF);
为什么两步缺一不可？
步骤①只是把引脚电气特性设好，但内部复用选择器还没拨过来，USART1 仍然无法驱动这个引脚。
步骤②只拨选择器，但引脚模式还是普通 GPIO，外设输出无法正常传出。
两步都做，PA9 才真正被 USART1 接管。
引脚方向复用功能电气配置
PA9TX（输出）AF7 = USART1_TX推挽、上拉、50MHz
PA10RX（输入）AF7 = USART1_RX推挽、上拉、50MHz

3硬件层 — USART1 外设配置

/* USART1 挂在 APB2 总线，时钟 90 MHz，必须先使能 */
RCC_APB2PeriphClockCmd(BSP_UART_CLK, ENABLE);

/* 帧格式：115200 / 8位数据 / 1位停止 / 无奇偶校验 = 标准 8N1 */
usart.USART_BaudRate            = 115200;
usart.USART_WordLength          = USART_WordLength_8b;
usart.USART_StopBits            = USART_StopBits_1;
usart.USART_Parity              = USART_Parity_No;
usart.USART_HardwareFlowControl = USART_HardwareFlowControl_None;
usart.USART_Mode                = USART_Mode_Rx | USART_Mode_Tx;
USART_Init(BSP_UART_PERIPH, &usart);

/* 每收到一个字节，RXNE 标志置位，触发中断 */
USART_ITConfig(BSP_UART_PERIPH, USART_IT_RXNE, ENABLE);

/* TX 寄存器空时，自动通知 DMA 填下一个字节（CPU 不参与） */
USART_DMACmd(BSP_UART_PERIPH, USART_DMAReq_Tx, ENABLE);

USART_Cmd(BSP_UART_PERIPH, ENABLE);

SPL 的 USART_Init() 会根据 APB2 时钟（90 MHz）和目标波特率自动计算 BRR（波特率寄存器）的值，无需手动计算。

RXNE 是什么？
RXNE = RX Not Empty。每当 USART1 接收移位寄存器把一个完整字节移入 DR 寄存器时，硬件自动将 RXNE 标志置 1，并触发 USART1_IRQHandler 中断。读取 DR 后 RXNE 自动清零。

4DMA 工作原理

4.1 为什么需要 DMA？

没有 DMA 时，CPU 发送 256 字节需要亲自搬运每一个字节：

/* 轮询发送（无 DMA）— CPU 被完全占用 */
for (i = 0; i < len; i++) {
    while (!(USART1->SR & USART_SR_TXE));  // 等 TX 寄存器空
    USART1->DR = buf[i];                    // 写一个字节
}

CPU 在循环里空转等待，这段时间内无法处理其他任何事情。

有了 DMA，CPU 只需下达指令，DMA 硬件自动完成搬运：

CPU：配置 DMA
启动传输
→
DMA 硬件
自动搬运
→
USART1 DR
逐字节发出
→
DMA TC 中断
通知 CPU 完成

4.2 STM32F4 DMA 结构

STM32F4 有两个 DMA 控制器，每个有 8 条 Stream，每条 Stream 有 8 个 Channel 可选：

DMA 控制器Stream 数量Channel 数量典型用途
DMA18（Stream0~7）每条 8 个APB1 外设（I2C、SPI、USART2~5 等）
DMA28（Stream0~7）每条 8 个APB2 外设（USART1/6、SPI1 等）+ SRAM
为什么必须用 DMA2 Stream7 Channel4？
每个外设的 DMA 请求信号只接在特定的 Stream + Channel 上，这是硬件固定连线，查数据手册《DMA request mapping》表才能确认。USART1_TX 固定对应 DMA2 Stream7 Channel4，不能选其他组合。

4.3 DMA 传输的三个要素

要素本项目配置含义
源地址（Memory）s_txBuf 的地址从哪里取数据，每次传完地址 +1
目标地址（Peripheral）&USART1->DR搬到哪里，地址固定不变
传输计数（NDTR）动态设置（= 发送长度）搬多少字节，每搬一个自动减 1，减到 0 停止

4.4 DMA 触发机制

DMA 不是一次性把所有字节瞬间搬完，而是由外设的"准备好"信号逐字节触发：

/* DMA 内部工作逻辑（示意，非真实代码）*/
while (NDTR > 0) {
    等待 USART1 发出 "TX 寄存器空" 请求信号;
    DR = s_txBuf[index++];   // 搬一个字节
    NDTR--;                  // 计数减一
}
触发 TC（Transfer Complete）中断;

USART1 每发完一个字节，就向 DMA 发出请求，DMA 立即填入下一个字节，如此循环，直到 NDTR 减到 0。

4.5 为什么 DMA 缓冲区不能放在 CCM？

CCM（0x1000_0000）DMA 绝对禁止访问
CCM（Core Coupled Memory）是 CPU 专属的紧耦合内存，只有 CPU 的 D-Code 总线能访问。DMA 控制器走的是 AHB 总线，根本看不到 CCM 地址空间。

如果把 s_txBuf 放在 CCM，DMA 读到的是垃圾数据，或直接触发 HardFault。

s_txBuf 是 static 全局变量，链接器默认放在 SRAM1（0x2000_0000），DMA 可以正常访问。

5硬件层 — DMA 配置

RCC_AHB1PeriphClockCmd(BSP_UART_DMA_CLK, ENABLE);  // DMA2 挂 AHB1
DMA_DeInit(BSP_UART_DMA_STREAM);                   // 复位，防止上电残留状态

dma.DMA_Channel            = BSP_UART_DMA_CHANNEL;           // Channel 4（硬件固定）
dma.DMA_PeripheralBaseAddr = (uint32_t)(&BSP_UART_PERIPH->DR); // 目标：USART1->DR
dma.DMA_Memory0BaseAddr    = (uint32_t)s_txBuf;               // 源：TX 缓冲区
dma.DMA_DIR                = DMA_DIR_MemoryToPeripheral;       // 内存 → 外设
dma.DMA_BufferSize         = 1U;                              // 初始占位，Transmit 时动态改
dma.DMA_PeripheralInc      = DMA_PeripheralInc_Disable;        // DR 地址固定
dma.DMA_MemoryInc          = DMA_MemoryInc_Enable;             // 内存地址每次 +1
dma.DMA_PeripheralDataSize = DMA_PeripheralDataSize_Byte;      // 按字节传
dma.DMA_MemoryDataSize     = DMA_MemoryDataSize_Byte;
dma.DMA_Mode               = DMA_Mode_Normal;                  // 传完即停，不循环
dma.DMA_Priority           = DMA_Priority_Low;
dma.DMA_FIFOMode           = DMA_FIFOMode_Disable;             // 直接模式，无 FIFO
DMA_Init(BSP_UART_DMA_STREAM, &dma);

/* 传输完成时触发中断，通知 CPU 可以发下一包 */
DMA_ITConfig(BSP_UART_DMA_STREAM, DMA_IT_TC, ENABLE);

6接收数据流：RXNE 中断 → 环形缓冲区

接收方向完全由中断驱动，CPU 主循环只需轮询读取缓冲区：

PA10 收到字节
→
USART1 RXNE
中断触发
→
ISR 回调
读 DR，写环形缓冲
→
主循环 Poll
读缓冲区处理

6.1 ISR 入口（stm32f4xx_it.c）

void USART1_IRQHandler(void)
{
    if (USART_GetITStatus(BSP_UART_PERIPH, USART_IT_RXNE) != RESET)
    {
        BSP_UART_RxISR_Callback();                                    // 交给 BSP 处理
        USART_ClearITPendingBit(BSP_UART_PERIPH, USART_IT_RXNE);    // 清中断标志
    }
}

6.2 BSP 回调（bsp_uart.c）

void BSP_UART_RxISR_Callback(void)
{
    uint8_t  byte     = (uint8_t)USART_ReceiveData(BSP_UART_PERIPH); // 读走字节，RXNE 自动清零
    uint16_t nextHead = (s_rxHead + 1U) % BSP_UART_RX_BUF_SIZE;

    if (nextHead != s_rxTail)   // 缓冲区未满
    {
        s_rxBuf[s_rxHead] = byte;
        s_rxHead = nextHead;
    }
    // 满了就丢弃新数据，保护旧数据不被覆盖
}
ISR 原则：最小操作
ISR 只做：读字节 → 写缓冲区 → 返回。绝对不能调用 printf、Delay_ms、malloc 等任何可能阻塞的函数，否则破坏实时性。

s_txDone = 1
ErrorStatus BSP_UART_Transmit(const uint8_t *pData, uint16_t len)
{
    /* 参数校验 */
    if (pData == NULL || len == 0U || len > BSP_UART_TX_BUF_SIZE)
        return ERROR;

    /* DMA 上一次传输尚未完成 → 返回 BUSY，由调用方下次重试 */
    if (s_txDone == 0U)
        return ERROR;

    s_txDone = 0U;   // 标记忙

    /* 把数据复制到 DMA 缓冲区（s_txBuf 在 SRAM1，DMA 可访问）*/
    uint16_t i;
    for (i = 0U; i < len; i++)
        s_txBuf[i] = pData[i];

    /* 重新配置 DMA 传输长度并启动 */
    DMA_Cmd(BSP_UART_DMA_STREAM, DISABLE);  // 必须先停才能修改 NDTR
    BSP_UART_DMA_STREAM->NDTR = len;         // 设置本次传输字节数
    DMA_Cmd(BSP_UART_DMA_STREAM, ENABLE);   // 启动，DMA 开始工作

    return SUCCESS;
}

DMA TX 完成中断

void DMA2_Stream7_IRQHandler(void)
{
    if (DMA_GetITStatus(BSP_UART_DMA_STREAM, DMA_IT_TCIF7) != RESET)
    {
        BSP_UART_TxDmaISR_Callback();                                      // s_txDone = 1
        DMA_ClearITPendingBit(BSP_UART_DMA_STREAM, DMA_IT_TCIF7);
    }
}
如果 DMA 未完成就来了新数据？
s_txDone == 0 时 BSP_UART_Transmit() 直接返回 ERROR（BUSY）。
App_Protocol_Poll() 拿到 ERROR 后丢弃这帧，等下次 Poll 再尝试。
阶段2 echo 测试中，DMA 传输速度远快于手动发送速度，实际极少触发。

8环形缓冲区详解

环形缓冲区（Ring Buffer）解决了 ISR 与主循环速度不匹配的问题：ISR 高速写入，主循环慢速读取，两者互不阻塞。

8.1 数据结构

static uint8_t  s_rxBuf[BSP_UART_RX_BUF_SIZE];       // 256 字节数组
static volatile uint16_t s_rxHead = 0U;  // ISR 写入位置（写指针）
static volatile uint16_t s_rxTail = 0U;  // 主循环读取位置（读指针）

8.2 工作原理图

  索引:  0   1   2   3   4   5   6   7  ...  255
         ┌───┬───┬───┬───┬───┬───┬───┬───┬───┬───┐
  数组:  │'H'│'e'│'l'│'l'│'o'│   │   │   │   │   │
         └───┴───┴───┴───┴───┴───┴───┴───┴───┴───┘
                              ↑               ↑
                           Head=5           Tail=0
                         （ISR 下次写这里）（主循环下次读这里）

  读取 'H' 后：Tail = 1
  ISR 再写 '!'：Head = 6

  Head == Tail 时：缓冲区为空
  (Head+1) % SIZE == Tail 时：缓冲区为满（牺牲一格判断满）

8.3 溢出策略：丢弃新数据

uint16_t nextHead = (s_rxHead + 1U) % BSP_UART_RX_BUF_SIZE;

if (nextHead != s_rxTail)   // 未满 → 写入
{
    s_rxBuf[s_rxHead] = byte;
    s_rxHead = nextHead;
}
// 满了 → 直接丢弃，已有数据不被覆盖

8.4 临界区保护

主循环读取 s_rxHead 时，需要防止 ISR 在读取中途修改它：

__disable_irq();    // 关中断，读取 head 快照
head = s_rxHead;
__enable_irq();     // 立刻恢复，临界区尽可能短
为什么用 volatile？
s_rxHead 和 s_rxTail 被 ISR 和主循环同时访问。volatile 告诉编译器每次都从内存读取，不允许缓存到寄存器，防止优化导致读到旧值。
