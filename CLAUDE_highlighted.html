1. 项目定位

本板为通用电机调试板，唯一职责：

PC 上位机
    ↕  UART 双向（指令下发 + 状态数据定期上报）
STM32F429ZG
    ↕  I2C
电机驱动 IC（型号不固定，均通过 I2C 通信）
  • STM32 不做闭环控制，无控制周期定时器需求
  • 功能：解析上位机指令 → 读写电机 IC 寄存器 → 将结果上报上位机
  • 电机 IC 可随时更换，切换只需修改 bsp_i2c.h 中的地址宏

3. 构建系统

3.1 工具链与环境

  • 开发环境：Windows，MSYS2/MinGW 原生（不使用 WSL）
  • 工具链：arm-none-eabi-gcc 12.x 或 13.x，已加入 MSYS2 PATH
  • Make：MSYS2 内置 GNU Make 4.x
  • 烧录：OpenOCD 0.12.x + ST-Link V2
  • 外设库：STM32F4xx 标准外设库 V1.8.0（SPL），禁止使用 HAL / LL

3.2 MSYS2 环境强制规则

  • 所有路径使用正斜杠 /，禁止反斜杠 \
  • 变量引用使用 $VAR，禁止 %VAR%
  • OpenOCD 通过 USB HID 识别 ST-Link，不依赖 COM 口编号

3.3 Make 目标

make              # Release 构建（-O2）
make DEBUG=1      # Debug 构建（-Og -g3 -DDEBUG）
make flash        # 编译并烧录
make debug_server # 启动 OpenOCD GDB server，端口 3333
make size         # 显示 Flash / RAM 占用
make clean        # 删除 build/

3.4 核心编译标志（不得省略）

CPU_FLAGS  = -mcpu=cortex-m4 -mfpu=fpv4-sp-d16 -mfloat-abi=hard -mthumb
OPT_FLAGS  = -ffunction-sections -fdata-sections -fno-common
WARN_FLAGS = -Wall -Wextra -Wshadow
LINK_FLAGS = -Wl,--gc-sections -specs=nano.specs

3.5 全局预定义宏

6. 代码规范

6.1 命名规范

/* 宏：全大写 + 下划线 */
#define BSP_UART_BAUDRATE_DEFAULT   115200U
#define PROTO_SOF                   0xAAU

/* 类型：PascalCase + _t */
typedef struct { uint8_t sof; uint8_t len; uint8_t cmd; uint8_t data[64]; uint8_t crc; } Proto_Frame_t;

/* BSP 函数：BSP_模块_动词 */
ErrorStatus BSP_UART_Init(void);
ErrorStatus BSP_I2C_WriteReg(uint8_t devAddr, uint8_t reg, uint8_t *pData, uint16_t len);
ErrorStatus BSP_I2C_Scan(uint8_t *pAddrList, uint8_t *pCount);

/* 静态/全局变量：s_ 前缀，ISR 共享加 volatile */
static volatile uint8_t s_uartRxFlag = 0U;

6.2 BSP 驱动层规则

  • 每个外设一对 bsp_xxx.h / bsp_xxx.c
  • 所有硬件参数只在 .h 宏中定义，.c 只引用宏，禁止裸端口/引脚数字
  • 初始化函数返回 ErrorStatus（SUCCESS / ERROR）
  • 驱动层禁止调用 printf()、Delay_ms()、业务逻辑

6.4 中断优先级配置

NVIC_PriorityGroupConfig(NVIC_PriorityGroup_4);  /* main() 第一行，只调用一次 */

6.5 ISR 规则

ISR 内绝对禁止：printf、Delay_ms、malloc/free、I2C 阻塞传输、任何 while 等待循环。

7. 各模块实现约定

7.1 UART

  • 接收：RXNE 中断逐字节写环形缓冲区，主循环解析完整帧
  • 发送：DMA 发送，发送前检查 s_uartTxDone
  • 支持动态修改波特率：9600 / 19200 / 38400 / 57600 / 115200 / 230400 / 460800 / 921600

7.2 I2C 通用驱动接口

ErrorStatus BSP_I2C_WriteReg(uint8_t devAddr, uint8_t reg, uint8_t *pData, uint16_t len);
ErrorStatus BSP_I2C_ReadReg(uint8_t devAddr, uint8_t reg, uint8_t *pData, uint16_t len);
ErrorStatus BSP_I2C_ReadRegs(uint8_t devAddr, uint8_t startReg, uint8_t *pData, uint16_t len);
ErrorStatus BSP_I2C_Scan(uint8_t *pAddrList, uint8_t *pCount);

所有 I2C 等待必须有超时（BSP_I2C_TIMEOUT），超时后自动调用 BSP_I2C_RecoverBus()（9 个 SCL 脉冲 + 软复位）。

7.3 上位机通信协议帧格式

[SOF 0xAA][LEN 1B][CMD 1B][DATA N B][CRC8 1B]
CRC8 覆盖：LEN + CMD + DATA
命令字值说明
CMD_PING0x01心跳，回 ACK + 固件版本
CMD_I2C_WRITE_REG0x10写电机 IC 寄存器
CMD_I2C_READ_REG0x11读单个寄存器
CMD_I2C_READ_REGS0x12批量读寄存器
CMD_I2C_SCAN0x13扫描总线设备
CMD_REPORT_STATUS0x20主动上报电机状态
CMD_SET_BAUDRATE0x30修改波特率
CMD_ACK0xF0成功应答
CMD_NACK0xF1失败应答 + 错误码

8. Claude Code 操作规范

  1. 完整读取本 CLAUDE.md

禁止原因

9.2 常见问题速查

现象优先排查
I2C BUSY 无法清除调用 BSP_I2C_RecoverBus()；检查上拉电阻
I2C 读回全 0xFF设备地址是否正确？是否左移 1 位？
I2C Scan 扫不到设备示波器确认波形；地址范围 0x08–0x77

UART 乱码波特率是否一致？HSE 是否 8 MHz？
DMA 缓冲区数据异常缓冲区是否在 CCM？→ 必须移至 SRAM1
编译报 undefined reference对应 .c 是否加入 Makefile C_SOURCES？
