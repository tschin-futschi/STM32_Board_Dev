1. UBOOT 协议帧结构

所有 ISP 命令通过同一种 12 字节帧封装（实际 I2C 发送 9 字节协议头 + N 字节 payload），结构如下：

[0]
checksum
[1]
ver
[2]
addr
[3]
module
[4]
event
[5]
len[0]
[6]
len[1]
[7]
ack
[8]
sum
[9..]
data
...
data
end
data
checksum = 字段[1..8] 累加和   |   ver = 协议版本 0x01   |   addr = 0x82 (CTL 控制帧) / 0x28 (ACK 应答帧)   |   module = HANK/SRAM/FLASH/END   |   event = 各模块下事件   |   len[2] = payload 字节长度（小端，实际只用 len[0]）   |   ack = 固定 0x01   |   sum = payload 累加和

SPI 模式下整帧前面再加一个 0x5A 起始字节。

7. 问题汇总（移植集成必须修正）

1 aw_reset_chip / aw_boot_control 是伪代码（isp.c L30, L57）——逐行翻译为 aw_i2c_write(0x6B 或 0x6F, ...) 调用，并区分两个 I2C 从机地址。
2 底层 I2C 四函数全空（isp.c L8, L14, L21, L26）——桥接到本项目 BSP_I2C2_WriteReg/ReadReg 与 BSP_GetTick。
3 aw_flash_jump_check 主体被注释（isp.c L181–L219）——恒返回 OK，跳转校验失效。
4 aw_i2c_isp_upload 缺 return（isp.c L177）+ 未在 .h 中声明。
5 两个 .h 用同一保护宏 __AW_UBOOT_REGISTER_H（bufbuilt.h L5 与 isp.h L5）——同时包含时第二个被吞。改 isp.h 的宏名为 __AW_UBOOT_ISP_H。
6 AW_U8 / AW_U32 类型未定义——必须在 main.h 或新建 aw_types.h 中补 typedef。
7 memcmp 校验前 10/14 字节包含 checksum/sum——应答端 sum 与写端通常不同，可能恒 NACK。建议改为只比较关键字段（module/event/ack/payload）。
8 aw_flash_block_erase_check 阻塞 25ms × len（isp.c L332）——可能打断本项目协议心跳，集成时需在专用 routine 内执行并暂停 TIM6 采样（参考 CLAUDE.md TODO 中的 H1）。
9 aw_uboot_buff_built 使用文件级静态 protocol_struct（bufbuilt.c L8）——非可重入。多线程或 ISR 调用会破坏帧。
10 data_len 字段 uint8 限制 payload 最大 255 字节——但写入命令最大 4 + 4*16 = 68 字节，未超限；读包最大 64 字节 + 4 地址 = 68 字节，也未超。设计裕量已用尽，不要再扩。
