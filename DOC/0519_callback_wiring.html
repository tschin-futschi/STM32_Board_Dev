5. 为什么这种「回调表」架构

多 IC 隔离：未来加 DW 系列芯片，DW 也有自己的 delay_ms / i2c_write / i2c_read。 如果 AW 与 DW 都把这些定义成全局 extern，链接期会撞名。 当前架构把它们都做成 static 私有 + 通过 aw_isp_ops_t（或将来 dw_isp_ops_t）暴露统一 API，跨 vendor 零符号冲突。
BSP 隔离：vendor 代码现在不 #include "bsp_i2c2.h"，意味着将来换 MCU / 换驱动 / 写单元测试时， vendor 代码完全不动，只换 aw_port_xxx.c 文件提供的实现即可。
