系统架构 (Core 框架层)

Core 是固件骨架：上电启动、向量表、时钟、NVIC 分组、外设初始化序列与主循环调度，全部 ISR 集中于 stm32f4xx_it.c。本身不含业务逻辑。

启动序列

上电 → Reset_Handler (.s) 拷贝 .data / 清 .bss → SystemInit (配 180MHz 时钟, VTOR=0x08000000) → __libc_init_array → main(): NVIC_PriorityGroup_4 ← 必须第一行 BSP_Tick_Init (SysTick 1ms) BSP_LED_Init BSP_UART_Init ✗→ HaltNoUart (100ms 快闪, 无串口) BSP_I2C1_Init ✗→ HaltOnInitFail(I2C1) BSP_I2C3_Init ✗→ ... (PMIC 总线, 必须先于 I2C2) BSP_PMIC_Init ✗→ 预设电压 BSP_PMIC_EnableSequence✗→ 开 LDO, 给电机 IC 上拉供电 BSP_PMIC_HwenInit (EXTI4 武装) BSP_I2C2_Init ✗→ 电机 IC 总线 (上拉已上电) aw_isp_init ✗→ 注册 ISP 回调 App_Protocol/Sample/FlashStore_Init → 发 PROTO_BOOT_OK → 进主循环
关键依赖：PMIC 必须先于 I2C2
电机 IC 的 SDA/SCL 上拉电阻由 PMIC 的 IOVDD 供电，因此初始化顺序严格为 I2C3 → PMIC_Init → PMIC_EnableSequence → I2C2。颠倒会导致 I2C2 总线无上拉而初始化失败。

主循环

while (1) {
    now = BSP_GetTick();
    App_Protocol_Poll();        // 解析 UART 环形缓冲里的帧并响应
    App_Sample_Poll();          // 组帧经 DMA 发 0xBB 数据流
    if (g_pmicHwenFlag) {       // HWEN 上升沿 → 重跑 PMIC 使能
        g_pmicHwenFlag = 0;
        BSP_PMIC_EnableSequence();
    }
    if (now - prevTick >= 500)  // 心跳 LED
        BSP_LED1_Toggle();
}
双通道故障指示
UART 可用时：发 0x0B BOOT_STATUS 帧（6 种 PROTO_INIT_FAIL_* 区分模块）+ 500ms 慢闪；UART 自身故障：退化为 100ms 快闪，现场可目视区分。

模块详情

Core 层是固件的框架骨架，负责上电启动、向量表、时钟、NVIC 分组、外设初始化序列与主循环调度，以及全部中断服务程序（ISR 集中于 stm32f4xx_it.c）。它本身不含业务逻辑，按依赖关系依次拉起 BSP/App，并在初始化失败时通过串口 0x0B BOOT_STATUS 帧 + LED 闪烁双通道指示故障。主循环以非阻塞轮询方式驱动协议解析、采样发送、PMIC HWEN 重配与 LED 心跳。内存布局通过链接脚本约束：固件仅占前 128KB Flash，余 896KB 留给 Flash 文件存储；堆为 0，栈 2KB，禁止动态分配。

公开 API

签名说明
int main(void)程序入口。先 NVIC_PriorityGroup_4，依次拉起各外设，任一失败进 Halt；成功后发 PROTO_BOOT_OK 并进主循环
static void HaltOnInitFail(Proto_BootStatus_t status)非 UART 模块初始化失败：发 0x0B 故障帧 + TxWait，再 500ms 慢闪 LED 死循环
static void HaltNoUart(void)UART 自身失败（诊断通道不可用）：100ms 快闪 LED 死循环，无串口帧
void SysTick_Handler(void)调 BSP_Tick_Increment，1ms 毫秒计数
void USART1_IRQHandler(void)RXNE 中断 → BSP_UART_RxISR_Callback 写环形缓冲 → 清标志
void DMA2_Stream7_IRQHandler(void)TX DMA 传输完成 → BSP_UART_TxDmaISR_Callback → 清 TCIF7
void TIM6_DAC_IRQHandler(void)采样定时器 Update → 清标志 → BSP_SampleTim_ISR_Callback（ISR 内做 I2C 采样，有意设计）
void EXTI4_IRQHandler(void)PMIC HWEN(PE4) 上升沿 → 清挂起 → 重采样过滤毛刺 → 置 g_pmicHwenFlag
void HardFault_Handler(void)裸寄存器开 GPIOF 时钟、配 PF13 输出，~50ms CPU 空转闪烁死循环

关键实现要点

初始化依赖顺序（核心正确性）

main.c:87-112 严格保证 I2C3→PMIC_Init→PMIC_EnableSequence→HwenInit→I2C2，注释明确 PMIC LDO 先于 I2C2 上电以给电机 IC 的 SDA/SCL 上拉供电。aw_isp_init(main.c:115) 在 I2C2+Tick 之后注册回调，顺序正确

NVIC 优先级分配与 CLAUDE.md 完全一致

Group_4（main.c:67）。UART RX=2(bsp_uart.h:47)、I2C EV/ER=5/6、TIM6=7(bsp_tim.h:22)、HWEN EXTI4=8(bsp_pmic.h:113)、UART TX DMA=10(bsp_uart.h:39)、SysTick=15。数值越小优先级越高，符合分组语义

向量表完整且与 ISR 一一对应

startup .s 中 EXTI4(157)/USART1(184)/TIM6_DAC(201)/DMA2_Stream7(217) 入口齐全；it.h 声明的 13 个 handler 均在向量表有对应 .word，未实现的全部弱别名指向 Default_Handler 自旋

时钟配置结论

system_stm32f4xx.c:364 PLL_M=8（HSE 8MHz→VCO输入1MHz），PLL_N=360/PLL_P=2 → SYSCLK 180MHz，与 CLAUDE.md 硬件规格吻合；VTOR=FLASH_BASE|0x00（:513），向量表在 0x08000000

内存布局防呆

ld:28 FLASH LENGTH 故意缩到 128K，固件增长越界会触发链接错误而非静默覆盖 [0x08020000,0x08100000) 的 Flash 文件存储区；_Min_Heap_Size=0 强约束无 malloc；_estack 取 SRAM1 顶

ISR 最小化

除 HardFault（故意死循环闪灯）外，所有 ISR 仅做 清标志+回调/置位，无阻塞调用，符合 ISR 规约。EXTI4 多一次 GPIO 重读做毛刺过滤（非阻塞，可接受）

启动失败双通道指示

UART 可用时走 0x0B BOOT_STATUS 帧（6 种 PROTO_INIT_FAIL_* 枚举区分模块，app_protocol.h:109-116）+ 500ms 慢闪；UART 自身故障退化为 100ms 快闪，现场可目视区分

数据流 / 控制流

流程
上电→Reset_Handler(.s:65) 拷贝.data/清.bss→SystemInit（配 180MHz 时钟、VTOR）→__libc_init_array→main()。main 内串行初始化各外设，任一返 !=SUCCESS 即跳 Halt 死循环。初始化完成后进主循环：每轮 BSP_GetTick 取时刻→App_Protocol_Poll（解析 UART 环形缓冲里的帧并响应）→App_Sample_Poll（组帧经 DMA 发 0xBB 数据流）→检查 g_pmicHwenFlag 决定是否重跑 PMIC 使能→按 500ms 翻转心跳 LED。中断侧：USART1 RX 填缓冲，TIM6 触发采样置标志/采集，DMA2_S7 完成发送回调，EXTI4 置 HWEN 标志，SysTick 累加毫秒。ISR 与主循环经 volatile 标志（g_pmicHwenFlag、s_sampleFlag、s_txDone、g_tick）单向通信。

模块耦合关系

Core 是顶层编排者，向下强依赖 BSP 全部模块（tick/led/uart/i2c1/i2c2/i2c3/pmic/tim）与 App 层（protocol/sample/flashstore）及 vendor AW 端口（aw_port_stm32 的 g_awOpsStm32 + aw_isp_init）。stm32f4xx_it.c 依赖 BSP 回调（BSP_UART_RxISR_Callback、BSP_UART_TxDmaISR_Callback、BSP_SampleTim_ISR_Callback、BSP_Tick_Increment）与 g_pmicHwenFlag。向量表符号名（TIM6_DAC_IRQHandler 等）与 startup .s 弱别名硬绑定。所有引脚/优先级宏来自 BSP/Inc，Core 不裸写端口（HardFault_Handler 除外，见 findings）。

本模块审查发现

中 无独立看门狗（IWDG），死循环/总线死锁无法自恢复
📍 Core/Src/main.c:142 (主循环) / 全局
问题：主循环与各 Halt 死循环均无看门狗喂狗机制。若主循环被某个 Poll 内的意外阻塞卡住（如 I2C 异常路径），或 fault handler 进入死循环，系统只能靠人工断电恢复。对电机调试板这类需长期挂机的设备，缺少 IWDG 自恢复是健壮性隐患。CLAUDE.md TODO 已列为结构性待办。
建议：评估引入 IWDG：在 BSP 增加 bsp_iwdg，主循环每轮喂狗（Halt 死循环可故意不喂以触发复位或保留人工诊断窗口，需与诊断需求权衡）。属新增功能，按规约应先与开发者确认。
低 HardFault_Handler 裸写端口寄存器位（魔法数字 + 绕过 BSP 分层）
📍 Core/Src/stm32f4xx_it.c:30-43
问题：用 (1U<<5) 直接置 RCC->AHB1ENR GPIOF 时钟位、(1U<<13)/(3U<<(13*2)) 直配 PF13。违反 CLAUDE.md「.c 中禁止裸写端口/引脚数字、必须引用宏」。虽然 HardFault 下故意不依赖 SPL/BSP（避免状态损坏时调库）有其合理性，但 13/5 这些位号是裸魔法数字，且与 BSP_LED 引脚定义重复，换 LED 引脚时此处会被漏改导致故障指示失效。CLAUDE.md TODO 已记录此项（HardFault 引脚魔法数字）。
建议：把 PF13/GPIOF 时钟位号定义为带注释的本地宏（如 HARDFAULT_LED_PORT_RCC_BIT / HARDFAULT_LED_PIN），或在注释中明确「故意独立于 BSP_LED 宏，换引脚需同步此处」，避免静默失效。
低 MemManage / BusFault / UsageFault 静默 while(1)，无任何指示
📍 Core/Src/stm32f4xx_it.c:47-60
问题：三个 fault handler 直接空死循环，既不闪灯也不发串口，现场无法与「正常运行卡死」区分。与 HardFault 有明确闪灯指示不一致。CLAUDE.md TODO 已列为低优先级。注：默认 SCB SHCSR 未使能这三个 fault 时它们会上升为 HardFault，但项目若启用 fpv4/MPU 仍可能直达此处。
建议：复用 HardFault_Handler 的闪灯逻辑（或让三者 b 到同一指示例程），保证任何 CPU fault 都有可见指示。
低 g_pmicHwenFlag 读-清-用非原子（理论上可丢一次边沿）
📍 Core/Src/main.c:154-158
问题：主循环 if(g_pmicHwenFlag){g_pmicHwenFlag=0; EnableSequence();}。若 EXTI4 在『读到非0』与『写0』之间又触发置1，则该次写0会清掉新边沿，丢失一次重配请求。flag 已正确声明 volatile，且 HWEN 边沿极低频、丢失最多延迟到下次真实边沿，实际影响可忽略，但严格讲是 ISR/主循环共享变量的 RMW 竞态。
建议：若需严格不丢边沿，可在 __disable_irq/__enable_irq 临界区内做读+清，或改用『先清后读硬件电平』模式。当前可保留并加注释说明可接受。
风格 主循环对 now 取一次后供多处使用，测试 Poll 用到偏旧时刻
📍 Core/Src/main.c:144-165
问题：now=BSP_GetTick() 在循环头取一次，随后 Protocol_Poll/Sample_Poll 可能耗时数百微秒至毫秒，Test_PMIC_PidRead_Poll(now) 与心跳判断使用的是循环开始的时刻。仅影响时间基准精度，且测试代码默认不编译；心跳用 >= 比较不会漏翻转。属轻微风格问题。
建议：如需更准可在心跳判断前重取 tick；当前实现可接受，无需改动。
风格 system_stm32f4xx.c 头部模板注释残留 25MHz / 168MHz 等非实际配置值
📍 Core/Src/system_stm32f4xx.c:35,47,59 等头部注释块
问题：文件头大段 SPL 模板注释列出 25MHz HSE、168MHz、PLL_M=25 等多套预设，与本项目实际生效的 HSE 8MHz/180MHz/PLL_M=8（:364/388/390）不符。实际宏定义正确，仅注释误导。该文件为 SPL 模板属只读复用范畴，按规约不逐行审，仅记录提醒。
建议：无需修改源（遵循『SPL 模板直接复用』规约）。如担心误导可在 CLAUDE.md/WIKI 注明『实际时钟以 :364 起 PLL 宏为准』。
