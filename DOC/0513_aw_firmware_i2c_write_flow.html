AW Firmware I2C 写指令（0x30）命令流

命令码 0x30 在本项目中统一称为 AW Firmware I2C 写指令， 专门服务 AW 系列电机 IC 烧录 SDK DLL（例如 AW86100.dll）的 pFunc_I2C_Write 回调。本文按链路顺序拆解从 DLL 调用到 I2C 总线落地的完整字节流。

1. 链路全貌

┌──────────────────────┐
│ 厂家 SDK DLL         │  pFunc_I2C_Write(DevId, WrSize, WrData)
│ (AW86100.dll 等)     │
└──────────┬───────────┘
           │ ① 回调参数
           ▼
┌──────────────────────┐
│ PC 上位机            │  按 0x30 协议帧封装，AddrSize 强制为 0
└──────────┬───────────┘
           │ ② UART 460800 8N1
           ▼
┌──────────────────────┐
│ STM32 (app_protocol) │  CRC 校验 → 命令分发 → BSP 透传写
└──────────┬───────────┘
           │ ③ 软件模拟 I2C（GPIO bit-bang，~870 kHz）
           ▼
┌──────────────────────┐
│ 电机 IC (DevId)      │
└──────────────────────┘

2. 阶段 ①：DLL 回调入口

2.1 回调签名

pFunc_I2C_Write(DevId, WrSize, WrData)
参数说明
DevId7-bit I2C 从机地址（不带 R/W 位，范围 0x00–0x7F）
WrSizeWrData 字节数
WrDataDLL 不透明字节流：DLL 自行把寄存器地址（如有）拼在头部，PC 和 STM32 都不允许解析

2.2 AW SDK 场景的关键约定（v2.4）

AW SDK 的 pFunc_I2C_Write 回调签名没有 AddrSize 参数。 DLL 自己决定是否在 WrData 头部拼上寄存器地址（1 / 2 字节），并将整块字节流交给上位机。 因此服务 AW SDK 时，0x30 协议帧中 AddrSize 恒为 0x00， Data 段就是 WrData 原文。

3. 阶段 ②：PC 端封装 0x30 协议帧

3.1 帧字节布局

┌──────┬──────┬─────┬──────┬─────┬───────┬──────────┬─────────┬───────────────┬──────┬──────┐
│ 0xAA │ 0x55 │ SEQ │ 0x30 │ LEN │ DevId │ AddrSize │ DataLen │   WrData...   │ CRCH │ CRCL │
│  1B  │  1B  │ 1B  │  1B  │ 1B  │  1B   │  =0x00   │ =WrSize │  WrSize 字节  │  1B  │  1B  │
└──────┴──────┴─────┴──────┴─────┴───────┴──────────┴─────────┴───────────────┴──────┴──────┘
                                                ↑
                          AddrSize=0 → AddrBytes 段长度为 0，整段跳过

3.2 字段说明

字段取值 / 计算
帧头0xAA 0x55 固定
SEQPC 端递增帧序号（0~255 循环）；重发时沿用原 SEQ
CMD0x30
LEN载荷字节数 = 1(DevId) + 1(AddrSize=0) + 1(DataLen) + WrSize = 3 + WrSize
DevId从 DLL 回调直接取，7-bit 不左移
AddrSize0x00（AW SDK 场景固定）
DataLen等于 WrSize
WrData[]DLL 字节流原文，WrData[0] 在前
CRC16多项式 0x8005、初值 0xFFFF、右移低位先行； 校验范围 = SEQ → WrData 末字节（不含帧头）；结果大端存放
注意：本协议使用 0x8005 + 右移算法，与标准 CRC16-MODBUS（反射多项式 0xA001）实现不同， 不能直接套用 Modbus CRC 库。

3.3 实例（写 4 字节字节流到 DevId=0x58）

参数值
DevId0x58
WrSize4
WrData0x01 0x23 0xA5 0x5A
LEN0x07（= 3 + 4）

完整帧（不含 CRC）：

AA 55 [SEQ] 30 07 58 00 04 01 23 A5 5A [CRCH] [CRCL]

4. 阶段 ③：UART 传输

  • 波特率：默认 460800，8 数据位 / 1 停止位 / 无校验 / 无流控
  • STM32 端 USART1 RXNE 中断把字节存入 256 B 环形缓冲
  • 主循环 App_Protocol_Poll() 完成帧定界（首字节 0xAA → 第二字节 0x55）与 CRC 校验

5. 阶段 ④：STM32 内部派发

App_Protocol_Poll()
   ├─ 帧头匹配 0xAA 0x55
   ├─ CRC16 校验 ────────► 失败：发 0x01 错误响应（错误码 0x01，SEQ 固定 0xFF）
   ├─ 命令码分发 → 0x30
   │     ├─ 解析 DevId / AddrSize(=0) / DataLen / WrData 指针
   │     └─ 调用 BSP I2C 透传写（AddrSize=0 走"裸字节流"分支）
   └─ I2C 返回 OK ───────► 同 CMD+SEQ、LEN=0 响应
      I2C NACK / 超时 ───► 0x01 错误响应（错误码 0x03 执行失败）
BSP 返回 ERROR 时应用层不做本地重试，直接向上位机回 NACK。 重试策略由 PC 端控制（100 ms 超时 + 最多 2 次重发，共 3 次）。

6. 阶段 ⑤：I2C2 总线时序

6.1 时序图（AddrSize == 0 单 transaction）

 SCL ─┐_┌─┐_┌─┐_┌─┐_ ... ─┐_┌─┐_
 SDA  S│ DevId<<1 + W │A│ WrData[0] │A│ WrData[1] │A│ ... │ WrData[N-1] │A│ P
      └───────────────┴─┴───────────┴─┴───────────┴─┴─────┴─────────────┴─┘
      START                                                              STOP

6.2 字节发送顺序

序号总线字节来源
START—STM32 产生起始条件
Byte 0DevId << 1（W 位 = 0）协议帧 DevId 字段
Byte 1WrData[0]协议帧 Data[0]
Byte 2WrData[1]协议帧 Data[1]
.........
Byte NWrData[N-1]协议帧 Data[N-1]
STOP—STM32 产生停止条件

6.3 STM32 端约束（来自 v2.4）

  • WrData[0] 先发，对应总线发送顺序，与协议帧载荷索引一致
  • 不得翻转字节序
  • 不得解析或重组 Data 段
  • 不得拆分 transaction，不得插入 RepeatedStart
  • 整个写操作在一次 START → STOP 之间完成

6.4 物理层参数

项目值
总线I2C2（PB10 SCL / PB11 SDA）
驱动方式软件模拟 I2C（GPIO bit-bang）
实测频率~870 kHz（bit 周期 ~1.15 μs）
单字节耗时参考地址 + 数据共 N+1 字节，单字节 ~9 bit ≈ 10.4 μs

7. 响应帧

结果响应字节说明
成功 AA 55 [SEQ] 30 00 [CRCH] [CRCL] LEN=0，SEQ 沿用请求帧
I2C 失败 AA 55 [SEQ] 01 01 03 [CRCH] [CRCL] 错误码 0x03（执行失败），SEQ 沿用请求帧
CRC 失败 AA 55 FF 01 01 01 [CRCH] [CRCL] 原 SEQ 不可信，SEQ 固定 0xFF，错误码 0x01

9. 与 0x21 写单寄存器的区别

维度0x21 写单寄存器0x30 AW Firmware I2C 写
目标设备固定 g_motorIcAddr任意 DevId（参数指定）
寄存器地址固定 16-bit 大端0 / 1 / 2 / 任意字节（AddrSize 指定）
数据宽度固定 2 字节任意长度（DataLen 指定）
字节解释协议层解析 reg / data 字段协议层完全不解析，原样透传
典型场景调试期常规寄存器读写烧录 SDK 回调、非标 I2C 设备

10. 要点速查

  • AW SDK 场景下 AddrSize 恒为 0，Data 段是 DLL 不透明字节流
  • STM32 端不得解析 / 重组 / 翻转 / 拆分 transaction
  • BSP I2C 失败时直接 NACK，不本地重试，重试由 PC 端控制（100ms × 3）
  • SEQ 重发不变，STM32 不去重，每次重新执行
  • CRC16 使用 0x8005 + 右移低位先行（非标准 MODBUS）
