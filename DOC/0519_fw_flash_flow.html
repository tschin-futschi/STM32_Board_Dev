8. 已知约束与 Known Issues

固件最大 64 KB：受 STM32 SRAM1 单缓冲限制。AW86006/86100 理论 max 是 128 KB，超出 64 KB 需要改成流式（边收边写 Flash），暂未实现。
0x34 EXEC 阻塞期 ~7s 不响应心跳：PC 端必须临时禁心跳检测或扩大超时。暂未实现可中断的 ISP 状态机。
真机未测试：整条链路架构通了，编译零错零警告，但没在 AW86xxx 实物上跑过。可能踩坑点：AW_CHIP_I2C_ADDR=0x6B 是从 vendor 伪代码推断、Wake 序列读回 0x01 的语义未经文档证实、I2C 总线时序（特别是 clock stretching）未验证等。
aw_flash_jump_check 是 stub：vendor 注释了真实现，当前函数体只 return ISP_OK。需要按 vendor 文档补回（涉及 END_FLASH_JUMP 协议帧）。
I2C2 总线独占：烧录期间会与采样 / 发生器（0x50/0x55/0x56/0x58）竞争 I2C2 总线。当前未做互斥保护（详见 CLAUDE.md TODO H1）。烧录前 PC 端应先停采样和发生器。
