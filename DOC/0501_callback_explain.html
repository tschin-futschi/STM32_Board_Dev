回调函数与回调注册

一、先理解普通函数调用

普通函数调用的关系是固定的：A 调用 B，写代码时就确定了。

void B(void)
{
    // 做某件事
}

void A(void)
{
    B();   // A 直接调用 B，编译时就确定了
}

这种方式的问题：如果 A 是底层模块（比如定时器 BSP），B 是上层模块（比如采样 App），那么 A 必须 #include B 的头文件，底层依赖了上层，这就是"反向依赖"，会导致代码耦合。

二、什么是回调函数

回调函数就是：你把一个函数交给别人，让别人在合适的时候替你调用它。

生活类比：

外卖类比

你点了外卖（注册回调），留了你的手机号（函数指针）。外卖到了（事件发生），骑手打你电话（调用回调）。你不需要一直盯着门口等（不需要轮询），骑手也不需要认识你（底层不依赖上层）——他只知道"到了就打这个号码"。

在 C 语言中，回调函数通过函数指针实现：

// 函数指针类型：指向一个 "无参数、无返回值" 的函数
typedef void (*Callback_t)(void);

// 一个变量，存储函数的地址
Callback_t  s_callback;

// 调用时，通过指针调用，效果等同于直接调用那个函数
s_callback();

三、什么是回调注册

回调注册就是：把你的函数地址告诉对方，存到对方的函数指针变量里。

"注册"这个词的意思 = "登记、记录下来"，以后需要的时候就调用它。

// 底层模块提供注册接口
void SetCallback(Callback_t cb)
{
    s_callback = cb;   // 把函数地址保存下来
}

// 上层模块注册自己的函数
SetCallback(MyFunction);   // "把 MyFunction 的地址登记到底层"

注册之后，底层模块在事件发生时就会调用你登记的函数，但底层完全不知道你的函数叫什么、做什么——它只知道"有人登记了一个函数，到时候调就行"。

四、直接调用 vs 回调：对比

直接调用（耦合）

// bsp_tim.c
#include "app_sample.h"  // ← 底层依赖上层！

void BSP_SampleTim_ISR_Callback(void)
{
    SampleTimerISR();  // ← 写死了
    s_sampleFlag = 1U;
}

BSP 层必须知道 App 层的函数名，换一个功能就要改 BSP 代码。

回调方式（解耦）

// bsp_tim.c
// 不需要 #include 任何 App 头文件

void BSP_SampleTim_ISR_Callback(void)
{
    if (s_callback != NULL)
        s_callback();  // ← 调谁？不知道，谁注册调谁
    s_sampleFlag = 1U;
}

BSP 层不知道也不关心上层是谁，上层通过注册告诉 BSP 该调什么。

五、当前项目中的实际代码

第 1 步：定义函数指针类型（bsp_tim.h）

// "回调函数长什么样"——无参数、无返回值的函数
typedef void (*BSP_SampleTim_Callback_t)(void);

第 2 步：底层提供注册接口 + 保存指针（bsp_tim.c）

// 存储回调的变量（初始为空）
static BSP_SampleTim_Callback_t s_callback = (void *)0;

// 注册接口：上层调用此函数，把自己的函数地址传进来
void BSP_SampleTim_SetCallback(BSP_SampleTim_Callback_t cb)
{
    s_callback = cb;   // 保存函数地址
}

// ISR 回调：先调用注册的函数，再置标志
void BSP_SampleTim_ISR_Callback(void)
{
    if (s_callback != (void *)0)
    {
        s_callback();   // 调用注册的函数（不知道它是谁）
    }
    s_sampleFlag = 1U;
}

第 3 步：上层注册自己的函数（app_sample.c）

// 这就是实际的回调函数——在 TIM6 ISR 中执行 I2C 读取
static void SampleTimerISR(void)
{
    // 发生器写入 + I2C 采样读取
    ...
}

// 初始化时注册：把 SampleTimerISR 的地址交给 BSP 层
void App_Sample_Init(void)
{
    ...
    BSP_SampleTim_SetCallback(SampleTimerISR);  // ← 注册！
}

注意

SampleTimerISR 没有加括号 ()。加括号是调用函数，不加括号是取函数地址。这里是把地址传给 BSP 层保存，不是立即调用。

七、为什么要用回调，不直接调用？

好处 说明
分层解耦 BSP 层不需要 #include App 层的头文件，符合 CLAUDE.md 的分层规范
可替换 将来如果换一个不同的采样逻辑，只需注册不同的函数，BSP 代码零修改
可选 如果没人注册回调（指针为空），ISR 就只置标志，行为和改造前完全一样
测试友好 测试时可以注册一个简单的测试函数，不需要修改任何正式代码
