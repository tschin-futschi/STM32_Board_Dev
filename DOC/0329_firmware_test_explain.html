STM32 固件测试脚本讲解

文件：firmware_test_i2C_command_0329.py 协议版本：v1.4 日期：2026-03-29 MCU：STM32F429ZGT6
1测试脚本概述
测试目的

通过 Python pyserial 与 STM32 建立 UART 连接，按照协议 v1.4 发送控制帧并验证响应，覆盖 UART 基础通信、系统控制命令、寄存器读写、采样功能四大模块的行为正确性。

PC（Python 脚本） │ UART 串口（COM3, 115200 bps 默认） │ 控制帧：AA 55 SEQ CMD LEN DATA CRC_H CRC_L │ 流帧 ：BB MASK LEN DATA... XOR ▼ STM32F429ZGT6 ← 被测固件 │ USART1 PA9/PA10 DMA TX + RX 中断 │ ├─ I2C2 → 电机 IC（地址由上位机设置，7-bit） ├─ I2C1 → INA 功耗测量 └─ I2C3 → PMIC RT5112WSC / GYRO
运行方式： 
# 仅 UART 基础测试（不需要电机）
python firmware_test_i2C_command_0329.py --port COM3

# 完整测试（含电机 I2C + 采样）
python firmware_test_i2C_command_0329.py --port COM3 --motor 0x20 --reg 0x0000

--portCOM3串口端口
--baud115200初始波特率
--motor0x00电机 IC 7-bit 地址；0x00 = 跳过电机/采样相关测试

数据流帧（STM32→PC，采样输出）
SOF0xBB
MASK0xXX
LENN
DATAN bytes
XOR0xXX

每通道 2 字节 int16 大端，XOR = MASK ^ LEN ^ 所有 DATA 字节逐字节异或。

已定义命令码
命令码名称方向数据段
0x00心跳双向空（LEN=0）
0x01错误响应STM32→PC1 字节错误码（0x01=CRC失败, 0x02=未知CMD, 0x03=执行失败）
0x02设置电机地址PC→STM321 字节（7-bit 地址）
0x03设置波特率PC→STM321 字节（波特率索引 0~7）
0x04系统复位PC→STM32空
0x05电机 PingPC→STM32空（测试 I2C ACK）
0x20读寄存器双向TX: 2B 地址; RX: 2B 值
0x21写寄存器双向4B（地址高低 + 值高低）
0x22批量读双向TX: 4B（起始地址+数量）; RX: 多包
0x50启动采样双向空
0x51停止采样双向空
0x52设置采样间隔双向1 字节（idx 0~7）
0x53设置通道掩码双向1 字节（bit0=ch0…bit7=ch7）
0x54设置寄存器映射双向16 字节（8 通道×2B 地址，0xFFFF=未映射）
3核心工具函数
crc16_modbus(data)

计算 CRC16-MODBUS，LSB-first，结果为 uint16。

def crc16_modbus(data):
    crc = 0xFFFF
    for byte in data:
        crc ^= byte
        for _ in range(8):
            crc = (crc >> 1) ^ 0x8005 \
                  if (crc & 1) else crc >> 1
    return crc
build_frame(seq, cmd, data)

构造完整控制帧 bytes，自动附加 CRC。

def build_frame(seq, cmd, data=b''):
    payload = bytes([seq, cmd, len(data)]) + data
    crc = crc16_modbus(payload)
    return b'\xAA\x55' + payload \
           + bytes([crc >> 8, crc & 0xFF])
recv_control_frame(ser, timeout)

逐字节状态机接收一个控制帧，自动跳过混入的 0xBB 流帧字节，超时返回 None。

→等待首字节 = 0xAA，否则跳过
→读第二字节；若 0xBB 则跳过流帧；若 0xAA 则继续等 0x55
→读 SEQ+CMD+LEN，再读 DATA+CRC_H+CRC_L
→调用 parse_frame() 返回字典 {seq, cmd, data, crc_ok}
recv_stream_frames(ser, count, timeout)

收集 N 个完整的 0xBB 流帧，返回字典列表，每项含 {mask, len, data, xor_ok}。

recv_bulk_packets(ser, total_pkts, timeout)

连续接收 total_pkts 个 CMD=0x22 的批量读响应包，每包含 [pktIdx][totalPkts][data…]。

_start_sampling / _stop_sampling

_start_sampling：依次发送 0x54（reg map）→ 0x53（mask）→ 0x52（interval）→ 0x50（start），每步均断言 ACK。
_stop_sampling：发送 0x51，等待响应，然后 flush RX 清空残余流帧。

4测试项详解
T2 UART 通信基础
测试项测试条件 / 操作预期结果
T2.2 正常心跳 发送 CMD=0x00，LEN=0，CRC 正确
  • 收到响应不为 None
  • 响应 CMD = 0x00
  • 响应 SEQ = 发送 SEQ（原样回显）
  • 响应 DATA 为空（LEN=0）
  • 响应 CRC 校验通过
T2.3 CRC 错误响应 故意发送 CRC=0xFFFF 的错误帧（正确 CRC ≠ 0xFFFF）
  • 收到错误响应帧（CMD=0x01）
  • 响应 SEQ = 0xFF（原帧不可信）
  • DATA = 0x01（错误码：CRC 校验失败）
  • 响应帧自身 CRC 通过
T2.4 未知命令码 发送 CMD=0x7F（协议未定义）
  • 收到错误响应帧（CMD=0x01）
  • 响应 SEQ = 原帧 SEQ（命令可识别）
  • DATA = 0x02（错误码：未知命令码）
T2.x 连续 3 帧 连续发送 3 帧心跳，不等待前帧响应直接发送下一帧
  • 每帧均能收到正确响应
  • 每帧响应 SEQ 与发送 SEQ 一致
  • 验证状态机在帧结束后正确复位，不影响下一帧解析

  • 设备离线 → CMD=0x01，DATA=0x03
T4.4 波特率切换 ① 发 CMD=0x03 + DATA=[0x03]（→57600）
② 收到 ACK 后关闭串口，以 57600 重新打开
③ 发心跳验证 57600 通信
④ 发 CMD=0x03 + DATA=[0x04]（→115200）
⑤ 收到 ACK 后关闭串口，以 115200 重新打开
⑥ 再次发心跳验证 115200 通信
  • 步骤 ①②③：57600 下心跳响应正常
  • 步骤 ④⑤⑥：切回 115200 后心跳响应正常
  • 使用 try/finally 保证无论失败都恢复 115200
T4.5 系统复位 ① 发 CMD=0x04
② 收到 ACK
③ 等待 1.5 s（含 PMIC 上电序列 4×10ms）
④ 发心跳验证 STM32 已重启
  • 步骤 ②：收到 CMD=0x04 ACK（ACK 必须在 NVIC_SystemReset() 前发完）
  • 步骤 ④：1.5 s 后心跳响应正常（CMD=0x00），表明固件已重新完成初始化
T4.5 复位后固件内的运行时状态（g_motorIcAddr 等）被清零，测试框架会在复位后重新执行 T4.1 设置电机地址。

6已知限制与注意事项
T5.5 采样中批量读（固件设计限制）

固件在发送采样流帧期间 UART TX DMA 处于 BUSY 状态，错误响应可能短暂延迟。脚本已加入最多 3 次 50ms 间隔重试以容忍此情况，但在极低波特率下仍有极小概率超时。

T6.3 高速采样帧率（CPU 限制）

idx=0 对应 100µs/帧（理论 10kHz）。PC 端接收/解析能力有限，实际收到的帧率可能低于固件发送帧率。若两端收帧不足 30 帧，比较逻辑会打印警告并跳过断言（不记为 FAIL）。

T4.4 波特率切换

切换后固件 ACK 必须在 UART_Cmd(DISABLE) 前完全发出（固件层已有 BSP_UART_TxWait() 保障）。脚本在收到 ACK 后等待 150ms 再重连，给固件留出切换时间。

T5.2 写寄存器破坏性

写入 0x1234 到测试寄存器后不会自动恢复原值。若测试寄存器为功能性寄存器（如 PMIC 控制寄存器），建议使用安全的只读寄存器地址进行测试，或在运行后手动复位板子。
