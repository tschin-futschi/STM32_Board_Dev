帧格式与解析流程

控制帧结构

byte 00xAA
byte 10x55
byte 2SEQ
byte 3CMD
byte 4LEN
byte 5..5+LEN-1DATA [n 字节]
byte 5+LENCRC_H
byte 6+LENCRC_L

CRC 校验范围：SEQ + CMD + LEN + DATA（不含帧头 0xAA 0x55），结果大端存放。

接收状态机（ProcessByte）

WAIT_SOF1
等 0xAA
→
WAIT_SOF2
等 0x55
→
WAIT_SEQ
记 seq，开始 CRC
→
WAIT_CMD
记 cmd，累积 CRC
→
WAIT_LEN
记 len，累积 CRC
→
WAIT_DATA
收 len 字节，累积 CRC
→
WAIT_CRC_H
记 CRC 高字节
→
WAIT_CRC_L
校验 → 派发
LEN=0 时跳过 WAIT_DATA 直接进入 WAIT_CRC_H。SOF2 不匹配时退回 WAIT_SOF1。
/* CRC 校验通过后派发，不通过回复错误帧 SEQ=0xFF */
case STATE_WAIT_CRC_L:
{
    uint16_t rxCrc = ((uint16_t)s_rxCrcHigh << 8U) | (uint16_t)byte;
    if (rxCrc != s_rxCrcAccum)
        SendErrorResp(PROTO_CRC_ERR_SEQ, PROTO_ERR_CRC_FAIL);
    else
        DispatchFrame(&s_rxFrame);
    s_state = STATE_WAIT_SOF1;
    break;
}

CRC16-MODBUS 算法

参数值
多项式0x8005
初值0xFFFF
位序低位先行（LSB-first）
结果存放大端（高字节在前）
校验范围SEQ + CMD + LEN + DATA
static uint16_t CRC16_Update(uint16_t crc, uint8_t byte)
{
    uint8_t bit;
    crc ^= (uint16_t)byte;
    for (bit = 0U; bit < 8U; bit++)
    {
        if ((crc & 0x0001U) != 0U)
            crc = (crc >> 1U) ^ 0x8005U;   /* LSB-first 反射多项式 */
        else
            crc >>= 1U;
    }
    return crc;
}

系统控制组（0x00 ~ 0x1F）

0x00 心跳（HEARTBEAT） 双向
字段值
PC → STM32 数据段空（LEN=0）
STM32 响应原样回复，CMD=0x00，数据段空
PC 发送周期每 1 s 一次
连接判断PC 连续 3 次无回复 → 判定断开
static void HandleHeartbeat(const Proto_Frame_t *pFrame)
{
    (void)SendFrame(pFrame->seq, (uint8_t)PROTO_CMD_HEARTBEAT, NULL, 0U);
}
0x01 错误响应（ERROR_RESP） STM32 → PC
错误码含义SEQ 填写规则
0x01CRC 校验失败固定填 0xFF（原帧 SEQ 不可信）
0x02未知命令码回传原帧 SEQ
0x03执行失败（如 I2C 无应答）回传原帧 SEQ
static void SendErrorResp(uint8_t seq, Proto_ErrCode_t err)
{
    uint8_t errByte = (uint8_t)err;
    (void)SendFrame(seq, (uint8_t)PROTO_CMD_ERROR_RESP, &errByte, 1U);
}
0x02 设置电机 IC I2C 地址（SET_MOTOR_ADDR） PC → STM32
字段值
数据段1 字节：7-bit I2C 地址（0x01 ~ 0x7F）
成功响应echo 原帧，数据段空
失败条件地址为 0x00 / 超出 7-bit 范围 / I2C 探测无应答
失败响应错误码 0x03
校验 addr 范围
0x01~0x7F
→
ProbeMotorIc(addr)
I2C ReadReg(reg=0x0000, 1字节)
→ ACK ✓
g_motorIcAddr = addr
回复 ACK
设置前会向目标地址发起 I2C 探测读（寄存器 0x0000，1 字节），确认设备存在后才更新 g_motorIcAddr。地址掉电不保存。
0x03 设置 UART 波特率（SET_BAUDRATE） PC → STM32
索引波特率索引波特率
0x0096000x04115200（默认）
0x01192000x05230400
0x02384000x06460800
0x03576000x07921600
顺序关键：STM32 先用当前（旧）波特率回复 ACK，再切换波特率。PC 需在收到 ACK 后同步切换。
/* 先回复，再切换 — PC 在收到 ACK 后切换 */
(void)SendFrame(pFrame->seq, (uint8_t)PROTO_CMD_SET_BAUDRATE, NULL, 0U);
BSP_UART_SetBaudrate(k_baudrateTable[idx]);
0x04 系统复位（RESET） PC → STM32
字段值
数据段空（LEN=0）
执行顺序先回复 ACK，再调用 NVIC_SystemReset()
复位后状态波特率恢复 115200，g_motorIcAddr 清零，采样停止
注意：复位前必须确保波特率已切回 115200，否则复位后 STM32 与 PC 波特率不匹配。
0x05 电机 IC Ping（MOTOR_PING） PC → STM32
字段值
数据段空（LEN=0）
成功响应echo 原帧
失败条件g_motorIcAddr == 0x00（未设置）或 I2C 无应答
复用 ProbeMotorIc(g_motorIcAddr)，向当前电机地址发送 I2C 探测读。可用于定期检查连接状态。
0x06 调试信息（DEBUG_INFO） STM32 → PC（主动上报）
字段值
数据段ASCII 字符串，最长 255 字节
SEQ固定为 0x00（非请求响应，不对应任何 PC 帧）
触发时机目前仅在启动采样失败时发送
/* 启动采样失败时同时发送调试信息 */
SendErrorResp(pFrame->seq, PROTO_ERR_EXEC_FAIL);
SendDebugInfo("Start failed: all channel reg maps are 0xFFFF");
调试信息为尽力发送，若 UART TX DMA 忙则丢弃，不影响主流程。

寄存器读写组（0x20 ~ 0x4F）

电机 IC 寄存器地址为 16-bit，所有命令中寄存器地址均以大端 2 字节传输。
0x20 读单个寄存器（READ_REG） PC → STM32
字段内容
请求数据段（LEN=2）[regH][regL] — 寄存器地址，大端
成功响应数据段（LEN=2）[valH][valL] — 寄存器值，大端
失败响应错误码 0x03
reg = ((uint16_t)pFrame->data[0] << 8U) | (uint16_t)pFrame->data[1];
App_Motor_ReadReg(reg, &val);   /* BSP_I2C2_ReadReg → 2 字节大端 */
resp[0] = (uint8_t)(val >> 8U);
resp[1] = (uint8_t)(val & 0xFFU);
0x21 写单个寄存器（WRITE_REG） PC → STM32
字段内容
请求数据段（LEN=4）[regH][regL][valH][valL] — 地址 + 数据，均大端
成功响应echo 原帧，数据段空
失败响应错误码 0x03
reg = ((uint16_t)pFrame->data[0] << 8U) | (uint16_t)pFrame->data[1];
val = ((uint16_t)pFrame->data[2] << 8U) | (uint16_t)pFrame->data[3];
App_Motor_WriteReg(reg, val);   /* BSP_I2C2_WriteReg → 2字节地址 + 2字节数据 */
0x22 批量读寄存器（BULK_READ） PC → STM32
字段内容
请求数据段（LEN=4）[startRegH][startRegL][cntH][cntL] — 起始地址 + 数量，均大端
数量范围1 ~ 32,130（超出返回错误码 0x03）
成功响应STM32 自动分包，每包最多 126 个寄存器，最多 255 包
失败条件数量为 0 或 > 32,130 / 采样进行中

响应包结构

SOF0xAA 0x55
SEQ原帧 SEQ
CMD0x22
LEN2+regs×2
DATA[0]pktIdx
DATA[1]totalPkts
DATA[2..N]寄存器数据（大端）
校验CRC16

分包逻辑

totalPkts = ceil(totalRegs / 126);   /* uint8_t，最大 255 */

for (pktIdx = 0; pktIdx < totalPkts; pktIdx++)
{
    regsThisPkt = min(totalRegs - regIdx, 126);
    /* 读取 regsThisPkt 个寄存器，起始地址 = startReg + regIdx */
    /* 打包发送，等待 TX DMA 空闲后发送下一包 */
    regIdx += regsThisPkt;
}
批量读期间主循环阻塞（在 Poll 函数内忙等 TX）。采样进行中禁止批量读。

数据段空（LEN=0）

0x51 停止采样（STOP_SAMPLE） PC → STM32

0x52 设置采样间隔（SET_INTERVAL） PC → STM32
索引间隔频率索引间隔频率

0x53 设置通道掩码（SET_CHANNEL） PC → STM32
字段值
数据段（LEN=1）1 字节掩码，bit0~bit7 对应通道 0~7，1=启用
失败条件掩码为 0x00
默认值0x01（仅通道 0）
掩码与寄存器映射共同决定有效通道（effectiveMask = channelMask & validRegMask）。
0x54 设置通道寄存器映射（SET_REG_MAP） PC → STM32
字段值
数据段（LEN=16）8 通道 × 2 字节寄存器地址，大端，通道 0 在最前
未映射哨兵值0xFFFF — 该通道不采样（注意：寄存器地址 0xFFFF 无法被映射）
默认值全部 0xFFFF（全部未映射）
/* 数据格式：[ch0_H][ch0_L][ch1_H][ch1_L]...[ch7_H][ch7_L] */
for (i = 0U; i < APP_SAMPLE_NUM_CHANNELS; i++)
{
    s_channelRegMap[i] = ((uint16_t)pData[i*2U] << 8U)
                       | (uint16_t)pData[i*2U + 1U];
}

数据流帧（0xBB）

byte 00xBB
byte 1effectiveMask
byte 2LEN
byte 3..3+LEN-1各通道数据（int16 大端）
byte 3+LENXOR 校验
字段说明
effectiveMask已启用且已映射的通道掩码（= channelMask & validRegMask）
LEN有效通道数 × 2（字节数）
通道数据每通道 2 字节 int16 大端，按通道编号从低到高排列
XOR 校验effectiveMask XOR LEN XOR 所有数据字节逐字节异或
触发源TIM6 UPDATE 中断 → ISR 置 s_sampleFlag → 主循环 App_Sample_Poll() 发帧
/* XOR 计算（app_sample.c）*/
xor = effectiveMask ^ len;          /* 初值：掩码 XOR LEN */
for (i = dataStart; i < idx; i++)
    xor ^= s_streamBuf[i];          /* XOR 所有数据字节 */
s_streamBuf[idx++] = xor;
