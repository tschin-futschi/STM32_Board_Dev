PMIC RT5112WSC 代码讲解

芯片：RT5112WSC  |  接口：I2C3（PA8/PC9） |  地址：0x20  |  日期：2026-03-27

1. 整体架构

PMIC 驱动分布在三个层次：

层次文件职责
BSP 传输层bsp_i2c3.h / bsp_i2c3.cI2C3 字节收发，不知道 PMIC 的存在
BSP 设备层bsp_pmic.h / bsp_pmic.cRT5112WSC 寄存器操作、上电时序、HWEN 中断
框架层Core/Src/main.c
Core/Src/stm32f4xx_it.c		初始化调用顺序、HWEN 中断服务程序

调用链示意：

main()
  → BSP_I2C3_Init()          // 建立 I2C3 传输通道
  → BSP_PMIC_Init()          // 设置电压 + 使能各路 LDO
  → BSP_PMIC_HwenInit()      // 开启 HWEN 中断监控

EXTI4_IRQHandler()
  → g_pmicHwenFlag = 1       // 仅置标志，不做任何阻塞操作

主循环
  → 检测 g_pmicHwenFlag
  → BSP_PMIC_Init()          // HWEN 重新触发时重新初始化

2. bsp_pmic.h — 寄存器与参数定义

2.1 I2C 地址

#define BSP_PMIC_I2C_ADDR    0x20U   /* 7-bit 地址，硬件固定 */

RT5112WSC 的 I2C 地址由芯片内部硬连线决定，固定为 0x20。调用 BSP_I2C3_WriteReg / BSP_I2C3_ReadReg 时，驱动会在地址末位附加读/写方向位（devAddr << 1），实际总线上出现的是 0x40（写）或 0x41（读）。

2.2 寄存器映射

宏名地址用途
BSP_PMIC_REG_BUCK_CTRL0x00BUCK 使能控制
BSP_PMIC_REG_LDO1_CTRL0x01LDO1 使能控制
BSP_PMIC_REG_LDO2_CTRL0x02LDO2 使能控制
BSP_PMIC_REG_LDO3_CTRL0x03LDO3 使能控制
BSP_PMIC_REG_LDO4_CTRL0x04LDO4 使能控制
BSP_PMIC_REG_PRODUCT_ID0x07芯片 ID（只读，用于自检）
BSP_PMIC_REG_BUCK_VOUT0x0CBUCK 输出电压设置
BSP_PMIC_REG_LDO1_VOUT0x0DLDO1 输出电压设置
BSP_PMIC_REG_LDO2_VOUT0x0ELDO2 输出电压设置
BSP_PMIC_REG_LDO3_VOUT0x0FLDO3 输出电压设置
BSP_PMIC_REG_LDO4_VOUT0x10LDO4 输出电压设置

2.3 控制寄存器位定义

#define BSP_PMIC_CTRL_ENABLE     0x8C   /* Bit7=1：使能输出；Bits[3:2]=0b11：模式位保持 */
#define BSP_PMIC_CTRL_DISABLE    0x0C   /* Bit7=0：关闭输出；Bits[3:2]=0b11：模式位保持 */

LDO_CTRL 寄存器的位布局：

Bit7   Bit6~4   Bit3  Bit2   Bit1  Bit0
 EN    保留      模式位[1]  模式位[0]   保留   保留
  1     0  0  0    1     1     0     0   →  0x8C（使能）
  0     0  0  0    1     1     0     0   →  0x0C（关闭）
读回验证时只检查 Bit7 是否为 1（tmp & 0x80U），因为寄存器中其他位可能存在只读状态位，严格比对 0x8C 可能误判失败。

2.4 电压公式

#define BSP_PMIC_VOLT_MIN_V    0.6f
#define BSP_PMIC_VOLT_MAX_V    3.775f
#define BSP_PMIC_VOLT_SCALE    80.0f

/* 寄存器值 = (目标电压 - 0.6) × 80，步进 12.5 mV */

示例计算：

目标电压计算过程寄存器值（十进制）寄存器值（十六进制）
1.80 V（LDO2）(1.8 - 0.6) × 80 = 96960x60
2.80 V（LDO1/LDO4）(2.8 - 0.6) × 80 = 1761760xB0
3.20 V（LDO3）(3.2 - 0.6) × 80 = 2082080xD0

2.5 HWEN 信号引脚

#define BSP_PMIC_HWEN_GPIO_PORT    GPIOE
#define BSP_PMIC_HWEN_PIN          GPIO_Pin_4      /* PE4 */
#define BSP_PMIC_HWEN_EXTI_LINE    EXTI_Line4
#define BSP_PMIC_HWEN_IRQn         EXTI4_IRQn
#define BSP_PMIC_HWEN_IRQ_PRIORITY 8U

HWEN（Hardware Enable）是 PMIC 的硬件使能信号，由外部电路（如 AP/SoC）控制。当 HWEN 从低变高时，PMIC 硬件上电，STM32 同时收到 EXTI4 中断通知，在主循环中重新执行软件初始化序列。

3. bsp_pmic.c — 函数详解

3.1 静态辅助函数

WaitMs — 阻塞等待

static void WaitMs(uint32_t ms)
{
    uint32_t start = BSP_GetTick();
    while ((BSP_GetTick() - start) < ms) { ; }
}

使用系统滴答计数器（SysTick，1ms 精度）做差值等待，不调用任何外部延时库。差值运算 BSP_GetTick() - start 在计数器溢出（约 49 天）时仍能正确工作。

WriteReg / ReadReg — I2C 单字节封装

static ErrorStatus WriteReg(uint8_t reg, uint8_t val)
{
    return BSP_I2C3_WriteReg(BSP_PMIC_I2C_ADDR, reg, &val, 1U);
}

static ErrorStatus ReadReg(uint8_t reg, uint8_t *pVal)
{
    return BSP_I2C3_ReadReg(BSP_PMIC_I2C_ADDR, reg, pVal, 1U);
}

将 I2C 地址固定为 BSP_PMIC_I2C_ADDR（0x20），上层代码只需关心寄存器地址和数据，不必每次传入设备地址。

3.2 BSP_PMIC_Init — 核心初始化

ErrorStatus BSP_PMIC_Init(void)
{
    uint8_t     tmp;
    ErrorStatus ret = SUCCESS;

    /* ── Step 1：预设各路输出电压（通道尚未使能）─────────────────── */
    if (BSP_PMIC_SetVout(BSP_PMIC_CH_LDO2, BSP_PMIC_DEFAULT_LDO2_V) != SUCCESS) { ret = ERROR; }
    if (BSP_PMIC_SetVout(BSP_PMIC_CH_LDO1, BSP_PMIC_DEFAULT_LDO1_V) != SUCCESS) { ret = ERROR; }
    if (BSP_PMIC_SetVout(BSP_PMIC_CH_LDO3, BSP_PMIC_DEFAULT_LDO3_V) != SUCCESS) { ret = ERROR; }
    if (BSP_PMIC_SetVout(BSP_PMIC_CH_LDO4, BSP_PMIC_DEFAULT_LDO4_V) != SUCCESS) { ret = ERROR; }
    if (ret != SUCCESS) { return ERROR; }

    /* ── Step 2：按顺序使能各路 LDO，步间延时 10 ms ──────────────── */
    if (BSP_PMIC_SetEnable(BSP_PMIC_CH_LDO2, ENABLE) != SUCCESS) { return ERROR; }
    WaitMs(BSP_PMIC_SEQ_DELAY_MS);   // 10 ms

    if (BSP_PMIC_SetEnable(BSP_PMIC_CH_LDO1, ENABLE) != SUCCESS) { return ERROR; }
    WaitMs(BSP_PMIC_SEQ_DELAY_MS);

    if (BSP_PMIC_SetEnable(BSP_PMIC_CH_LDO3, ENABLE) != SUCCESS) { return ERROR; }
    WaitMs(BSP_PMIC_SEQ_DELAY_MS);

    if (BSP_PMIC_SetEnable(BSP_PMIC_CH_LDO4, ENABLE) != SUCCESS) { return ERROR; }

    /* ── Step 3：读回验证——确认 Bit7（使能位）已置 1 ─────────────── */
    if (ReadReg(BSP_PMIC_REG_LDO1_CTRL, &tmp) != SUCCESS) { return ERROR; }
    if ((tmp & 0x80U) == 0U) { return ERROR; }

    if (ReadReg(BSP_PMIC_REG_LDO2_CTRL, &tmp) != SUCCESS) { return ERROR; }
    if ((tmp & 0x80U) == 0U) { return ERROR; }

    if (ReadReg(BSP_PMIC_REG_LDO3_CTRL, &tmp) != SUCCESS) { return ERROR; }
    if ((tmp & 0x80U) == 0U) { return ERROR; }

    if (ReadReg(BSP_PMIC_REG_LDO4_CTRL, &tmp) != SUCCESS) { return ERROR; }
    if ((tmp & 0x80U) == 0U) { return ERROR; }

    return SUCCESS;
}

三个步骤的设计理由：

步骤目的为何先设电压再使能
Step 1：预设电压写入 VOUT 寄存器，但输出尚未开启防止使能瞬间输出错误电压，损坏负载
Step 2：顺序使能LDO2→LDO1→LDO3→LDO4，步间 10ms让每路电源充分稳定后再上下一路，避免浪涌
Step 3：读回验证回读 CTRL 寄存器 Bit7确认 I2C 写入成功，芯片实际执行了使能
Step 1 使用 ret 累积所有 SetVout 失败，四路全部尝试后再统一返回错误，避免某一路失败时跳过其余路的设置。Step 2 和 Step 3 则遇到第一个失败立即返回。

3.3 BSP_PMIC_SetVout — 设置输出电压

ErrorStatus BSP_PMIC_SetVout(uint8_t ch, float voltV)
{
    uint8_t regAddr;
    uint8_t regVal;

    /* 范围检查（闭区间：0.6 V ≤ voltV ≤ 3.775 V）*/
    if (!((voltV >= BSP_PMIC_VOLT_MIN_V) && (voltV <= BSP_PMIC_VOLT_MAX_V)))
        return ERROR;

    /* 电压 → 寄存器值转换（四舍五入）*/
    regVal = (uint8_t)((voltV - BSP_PMIC_VOLT_MIN_V) * BSP_PMIC_VOLT_SCALE + 0.5f);

    /* 通道 → 寄存器地址映射 */
    switch (ch)
    {
        case BSP_PMIC_CH_BUCK: regAddr = BSP_PMIC_REG_BUCK_VOUT; break;
        case BSP_PMIC_CH_LDO1: regAddr = BSP_PMIC_REG_LDO1_VOUT; break;
        case BSP_PMIC_CH_LDO2: regAddr = BSP_PMIC_REG_LDO2_VOUT; break;
        case BSP_PMIC_CH_LDO3: regAddr = BSP_PMIC_REG_LDO3_VOUT; break;
        case BSP_PMIC_CH_LDO4: regAddr = BSP_PMIC_REG_LDO4_VOUT; break;
        default: return ERROR;
    }

    return WriteReg(regAddr, regVal);
}

关键点：

  • 加 0.5f 实现四舍五入，避免浮点截断导致实际电压偏低 12.5mV
  • 只写 VOUT 寄存器，不使能通道——使能由 BSP_PMIC_SetEnable 单独控制

3.4 BSP_PMIC_SetEnable — 使能/关闭通道

ErrorStatus BSP_PMIC_SetEnable(uint8_t ch, FunctionalState state)
{
    uint8_t regAddr;
    uint8_t val = (state == ENABLE) ? BSP_PMIC_CTRL_ENABLE : BSP_PMIC_CTRL_DISABLE;
    /*                                     0x8C                    0x0C          */

    switch (ch)
    {
        case BSP_PMIC_CH_BUCK: regAddr = BSP_PMIC_REG_BUCK_CTRL; break;
        case BSP_PMIC_CH_LDO1: regAddr = BSP_PMIC_REG_LDO1_CTRL; break;
        case BSP_PMIC_CH_LDO2: regAddr = BSP_PMIC_REG_LDO2_CTRL; break;
        case BSP_PMIC_CH_LDO3: regAddr = BSP_PMIC_REG_LDO3_CTRL; break;
        case BSP_PMIC_CH_LDO4: regAddr = BSP_PMIC_REG_LDO4_CTRL; break;
        default: return ERROR;
    }

    return WriteReg(regAddr, val);
}

3.5 BSP_PMIC_ReadPid — 读取芯片 ID

ErrorStatus BSP_PMIC_ReadPid(uint8_t *pPid)
{
    return ReadReg(BSP_PMIC_REG_PRODUCT_ID, pPid);
}

读取 0x07 寄存器的产品 ID，用于启动时验证 I2C 通信是否正常、芯片型号是否匹配。I2C 失败时返回 ERROR，调用方可据此判断通信状态。

3.6 BSP_PMIC_HwenInit — HWEN 中断初始化

void BSP_PMIC_HwenInit(void)
{
    /* 1. 使能时钟：GPIOE（AHB1）和 SYSCFG（APB2）*/
    RCC_AHB1PeriphClockCmd(BSP_PMIC_HWEN_GPIO_CLK, ENABLE);
    RCC_APB2PeriphClockCmd(RCC_APB2Periph_SYSCFG,   ENABLE);

    /* 2. PE4 配置为浮空输入（外部电路已定义空闲电平）*/
    gpioInit.GPIO_Pin  = BSP_PMIC_HWEN_PIN;
    gpioInit.GPIO_Mode = GPIO_Mode_IN;
    gpioInit.GPIO_PuPd = GPIO_PuPd_NOPULL;
    GPIO_Init(BSP_PMIC_HWEN_GPIO_PORT, &gpioInit);

    /* 3. SYSCFG：将 GPIOE Pin4 路由到 EXTI Line4 */
    SYSCFG_EXTILineConfig(BSP_PMIC_HWEN_EXTI_PORT, BSP_PMIC_HWEN_EXTI_PIN);

    /* 4. EXTI：上升沿触发，中断模式 */
    extiInit.EXTI_Line    = BSP_PMIC_HWEN_EXTI_LINE;
    extiInit.EXTI_Trigger = EXTI_Trigger_Rising;   // HWEN 低→高时触发
    extiInit.EXTI_Mode    = EXTI_Mode_Interrupt;
    extiInit.EXTI_LineCmd = ENABLE;
    EXTI_Init(&extiInit);

    /* 5. NVIC：优先级 8（低于 UART/I2C，高于 SysTick）*/
    nvicInit.NVIC_IRQChannel                   = BSP_PMIC_HWEN_IRQn;
    nvicInit.NVIC_IRQChannelPreemptionPriority = BSP_PMIC_HWEN_IRQ_PRIORITY;  // 8
    nvicInit.NVIC_IRQChannelSubPriority        = 0U;   // Group_4：子优先级无效
    nvicInit.NVIC_IRQChannelCmd                = ENABLE;
    NVIC_Init(&nvicInit);
}
此函数必须在 BSP_PMIC_Init() 之后调用。若在 PMIC 初始化完成前开启 HWEN 中断，上电时的 HWEN 上升沿会触发中断，导致 BSP_PMIC_Init() 被重复调用。

4. HWEN 中断服务程序

位于 Core/Src/stm32f4xx_it.c：

void EXTI4_IRQHandler(void)
{
    if (EXTI_GetITStatus(BSP_PMIC_HWEN_EXTI_LINE) != RESET)
    {
        EXTI_ClearITPendingBit(BSP_PMIC_HWEN_EXTI_LINE);  // 先清标志，防止重入

        /* 读取 GPIO 电平确认 HWEN 仍为高（简单去抖）*/
        if (GPIO_ReadInputDataBit(BSP_PMIC_HWEN_GPIO_PORT, BSP_PMIC_HWEN_PIN) != RESET)
        {
            g_pmicHwenFlag = 1U;   // 仅置标志，所有重初始化工作交给主循环
        }
    }
}

ISR 遵循"最小操作"原则：

  • 先清中断标志，再读 GPIO，防止在读取期间新边沿到来时丢失
  • 读 GPIO 电平做去抖：若已回落为低，说明是毛刺，不置标志
  • 不调用任何阻塞函数（无 DelayMs、无 I2C 操作）

5. main.c 中的集成

5.1 初始化顺序

BSP_Tick_Init();       // SysTick — WaitMs 依赖此模块，必须最先初始化
BSP_LED_Init();
BSP_UART_Init();

BSP_I2C1_Init();       // 电机 IC 通道
BSP_I2C3_Init();       // PMIC 通道 — 必须在 PMIC_Init 之前

BSP_PMIC_Init();       // 上电时序：预设电压 → 顺序使能 → 读回验证
BSP_PMIC_HwenInit();   // HWEN 中断 — 必须在 PMIC_Init 之后

App_Protocol_Init();   // 协议层 — PMIC 供电正常后才启动上层应用
若 BSP_I2C3_Init() 或 BSP_PMIC_Init() 返回 ERROR，程序立即进入 while(1) 死循环。此时 LED 不再闪烁，可直观判断故障发生在哪个阶段。

5.2 主循环中的 HWEN 处理

while (1)
{
    App_Protocol_Poll();

    if (g_pmicHwenFlag != 0U)          // ISR 置位
    {
        g_pmicHwenFlag = 0U;            // 先清标志
        (void)BSP_PMIC_Init();            // 重新执行完整上电时序
    }

    /* LED 心跳 */
}

主循环中的 BSP_PMIC_Init() 调用包含约 30ms 的阻塞（3 次 WaitMs(10ms)）。由于串口协议超时为 100ms，30ms 阻塞在容限内，不会触发上位机超时重传。
