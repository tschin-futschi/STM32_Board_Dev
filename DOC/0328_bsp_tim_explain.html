bsp_tim 定时器设置讲解

目标：让 TIM6 以可配置的频率（100 Hz ～ 10000 Hz）周期性触发中断，每次中断只置一个标志位，供主循环采样使用。

① 时钟来源

TIM6 挂在哪条总线？

STM32F429 的定时器分布在两条总线上：

总线频率定时器
APB145 MHzTIM2~TIM7, TIM12~TIM14
APB290 MHzTIM1, TIM8~TIM11

TIM6 挂在 APB1（45 MHz）。但 STM32F4 有一个规则：

当 APB1 预分频系数 ≠ 1 时，挂在 APB1 上的定时器时钟 = APB1 频率 × 2。
本项目 APB1 预分频 = 4（180 MHz / 4 = 45 MHz），所以 TIM6 实际时钟 = 45 × 2 = 90 MHz。
/* bsp_tim.c 中使能 TIM6 时钟 */
RCC_APB1PeriphClockCmd(BSP_SAMPLE_TIM_RCC_APB1, ENABLE);

/* bsp_tim.h 中的宏定义 */
#define BSP_SAMPLE_TIM_RCC_APB1   RCC_APB1Periph_TIM6

② PSC 预分频寄存器

把 90 MHz 降到 1 MHz

TIM6 内部有一个 16-bit 预分频器（PSC），实际分频系数 = PSC + 1：

计数器时钟 = TIM6 输入时钟 / (PSC + 1) = 90 MHz / (89 + 1) = 1 MHz

1 MHz 意味着计数器每 1 μs 加一次，非常方便直接用微秒计算间隔。

/* bsp_tim.h */
#define BSP_SAMPLE_TIM_PSC   89U

/* bsp_tim.c — 写入 TIM_Prescaler */
timInit.TIM_Prescaler = BSP_SAMPLE_TIM_PSC;  /* 89 → 分频系数 90 */

③ ARR 自动重载寄存器

决定中断触发周期

计数器从 0 开始向上计数，计到 ARR 时溢出并触发更新中断，然后重新从 0 开始：

中断周期 = (ARR + 1) 个计数 = (ARR + 1) μs
中断频率 = 1 MHz / (ARR + 1) Hz

因此 ARR = 采样间隔（μs）− 1：

索引采样间隔频率ARR 值
0100 μs10000 Hz99
1200 μs5000 Hz199
2300 μs3333 Hz299
3500 μs2000 Hz499
4750 μs1333 Hz749
51000 μs（默认）1000 Hz999
61500 μs667 Hz1499
72000 μs500 Hz1999
/* bsp_tim.c — ARR 查表 */
static const uint16_t k_arrTable[BSP_SAMPLE_TIM_IDX_MAX + 1U] =
{
     99U,  /* idx 0: 100 us */
    199U,  /* idx 1: 200 us */
    299U,  /* idx 2: 300 us */
    499U,  /* idx 3: 500 us */
    749U,  /* idx 4: 750 us */
    999U,  /* idx 5: 1000 us (default) */
   1499U, /* idx 6: 1500 us */
   1999U, /* idx 7: 2000 us */
};

④ BSP_SampleTim_Init()

初始化流程（逐行）

ErrorStatus BSP_SampleTim_Init(void)
{
    TIM_TimeBaseInitTypeDef timInit;
    NVIC_InitTypeDef        nvicInit;

    /* 1. 使能 TIM6 的 APB1 总线时钟
          不使能时钟，任何寄存器写入都无效 */
    RCC_APB1PeriphClockCmd(BSP_SAMPLE_TIM_RCC_APB1, ENABLE);

    /* 2. 填充初始化结构体 */
    TIM_TimeBaseStructInit(&timInit);           /* 先清零，防止残留值 */
    timInit.TIM_Prescaler    = BSP_SAMPLE_TIM_PSC;              /* PSC=89 → 1MHz */
    timInit.TIM_Period       = k_arrTable[BSP_SAMPLE_TIM_DEFAULT_IDX]; /* ARR=999 → 1ms */
    timInit.TIM_CounterMode = TIM_CounterMode_Up;              /* 向上计数 */
    timInit.TIM_ClockDivision = TIM_CKD_DIV1;                 /* 不分频滤波时钟 */
    TIM_TimeBaseInit(BSP_SAMPLE_TIM_PERIPH, &timInit);

    /* 3. 清除 Init 时可能产生的更新标志
          TIM_TimeBaseInit 写 PSC 会触发一次 UG 事件，需手动清除
          否则 Start() 后立即触发一次"假"中断 */
    TIM_ClearFlag(BSP_SAMPLE_TIM_PERIPH, TIM_FLAG_Update);

    /* 4. 配置 NVIC：优先级 7，使能 TIM6_DAC 中断通道 */
    nvicInit.NVIC_IRQChannel                   = BSP_SAMPLE_TIM_IRQn;  /* TIM6_DAC_IRQn */
    nvicInit.NVIC_IRQChannelPreemptionPriority = BSP_SAMPLE_TIM_IRQ_PRIORITY; /* 7 */
    nvicInit.NVIC_IRQChannelSubPriority        = 0U;
    nvicInit.NVIC_IRQChannelCmd                = ENABLE;
    NVIC_Init(&nvicInit);

    s_sampleFlag = 0U;
    return SUCCESS;
}
注意：Init() 结束后定时器不运行，计数器停在 0，没有中断。必须显式调用 BSP_SampleTim_Start() 才开始计时。这样可以先配置好采样参数，再一键启动。

为什么要清除 TIM_FLAG_Update？
SPL 的 TIM_TimeBaseInit() 在写 PSC 寄存器时，会向 EGR 寄存器写 UG 位以立即更新影子寄存器，这会产生一个更新事件，导致 Update 标志被置位。如果不清除，调用 Start() 使能中断后会立刻触发一次"假"中断，第一个采样点时序不对。

⑤ Start() / Stop()

void BSP_SampleTim_Start(void)
{
    TIM_ClearFlag(BSP_SAMPLE_TIM_PERIPH, TIM_FLAG_Update); /* 再次清标志，防止 Stop→Start 期间残留 */
    s_sampleFlag = 0U;                                        /* 清软件标志 */
    TIM_ITConfig(BSP_SAMPLE_TIM_PERIPH, TIM_IT_Update, ENABLE);  /* 使能更新中断 */
    TIM_Cmd(BSP_SAMPLE_TIM_PERIPH, ENABLE);                    /* 启动计数器 */
}

void BSP_SampleTim_Stop(void)
{
    TIM_Cmd(BSP_SAMPLE_TIM_PERIPH, DISABLE);                    /* 停止计数器 */
    TIM_ITConfig(BSP_SAMPLE_TIM_PERIPH, TIM_IT_Update, DISABLE); /* 禁用更新中断 */
    TIM_ClearFlag(BSP_SAMPLE_TIM_PERIPH, TIM_FLAG_Update);      /* 清除硬件标志 */
    s_sampleFlag = 0U;                                           /* 清软件标志 */
}

Stop() 的操作顺序说明：

步骤操作原因
1TIM_Cmd DISABLE计数器立即停止，不再产生新的溢出事件
2TIM_ITConfig DISABLE禁用更新中断，NVIC 不再响应 TIM6 的请求
3TIM_ClearFlag清除停止瞬间可能残留的 Update 标志
4s_sampleFlag = 0丢弃主循环尚未处理的采样请求

⑥ BSP_SampleTim_SetFreq() — 动态改频率

ErrorStatus BSP_SampleTim_SetFreq(uint8_t idx)
{
    if (idx > BSP_SAMPLE_TIM_IDX_MAX)   /* 索引越界检查 */
    {
        return ERROR;
    }
    TIM_SetAutoreload(BSP_SAMPLE_TIM_PERIPH, k_arrTable[idx]);  /* 只改 ARR */
    return SUCCESS;
}

为什么只改 ARR 就够了？

PSC 决定计数速度（固定 1 MHz），ARR 决定溢出点。改变 ARR 等于改变溢出周期，即改变采样频率。TIM_SetAutoreload() 直接写 ARR 寄存器，无需停止定时器，运行中实时生效（当前计数周期结束后新 ARR 生效）。

TIM6 没有预装载寄存器（ARR preload）默认禁用，写入立即生效。若当前计数值已超过新 ARR，会在下一个计数器溢出时才更新，不会丢失中断。

⑦ ISR 中断服务程序

stm32f4xx_it.c 中的 ISR

/* stm32f4xx_it.c */
void TIM6_DAC_IRQHandler(void)
{
    if (TIM_GetITStatus(BSP_SAMPLE_TIM_PERIPH, TIM_IT_Update) != RESET)
    {
        TIM_ClearITPendingBit(BSP_SAMPLE_TIM_PERIPH, TIM_IT_Update); /* 必须先清标志 */
        BSP_SampleTim_ISR_Callback();                                  /* 再置软件标志 */
    }
}

为什么 ISR 名是 TIM6_DAC_IRQHandler？

TIM6 和 DAC 共享同一个 NVIC 中断向量（IRQ54），SPL 和启动文件都用 TIM6_DAC_IRQHandler 作为向量名。本项目不用 DAC，所以这个 ISR 实际上只处理 TIM6 的更新事件。

为什么先清标志再置软件标志？

必须在 ISR 内清除硬件中断标志，否则 ISR 退出后 NVIC 发现标志仍置位，会立即再次进入 ISR，形成死循环。清除之后再调用 Callback 置软件标志，顺序不能颠倒。

bsp_tim.c 中的 Callback

/* bsp_tim.c */
void BSP_SampleTim_ISR_Callback(void)
{
    s_sampleFlag = 1U;   /* 唯一操作：置标志，ISR 立即退出 */
}

ISR 总耗时约 3～5 个指令周期（< 100 ns），对系统实时性影响可忽略。

⑧ 标志位机制

/* bsp_tim.c */
static volatile uint8_t s_sampleFlag;   /* volatile：防止编译器优化掉 ISR 的写入 */

uint8_t BSP_SampleTim_GetFlag(void)  { return s_sampleFlag; }
void    BSP_SampleTim_ClearFlag(void) { s_sampleFlag = 0U; }

volatile 的作用：
没有 volatile，编译器可能把 s_sampleFlag 缓存在寄存器里，主循环永远读到缓存值 0，看不到 ISR 写入的 1。加了 volatile 后，每次读取都强制从内存地址读，保证看到 ISR 的修改。

主循环使用方式（4.4 阶段 App_Sample_Poll 将这样调用）：

void App_Sample_Poll(void)
{
    if (BSP_SampleTim_GetFlag() == 0U) { return; }  /* 没有新采样请求，直接返回 */
    BSP_SampleTim_ClearFlag();                       /* 先清标志，再采样，防止漏采 */

    /* 读寄存器、组帧、发送数据流帧 ... */
}
