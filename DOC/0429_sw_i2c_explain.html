0. 整体架构

整个驱动从底到顶分为 5 层，每一层只依赖下一层：

2. DWT Cycle Counter — 精确延时引擎

这是整个驱动的时序基础。DWT（Data Watchpoint and Trace）是 Cortex-M4 内核自带的调试硬件，其中的 CYCCNT 是一个 32-bit 自由运行计数器，每个 CPU 时钟周期 +1。

32static void DWT_Init(void)
33{
34    CoreDebug->DEMCR |= CoreDebug_DEMCR_TRCENA_Msk;
35    DWT->CYCCNT = 0U;
36    DWT->CTRL  |= DWT_CTRL_CYCCNTENA_Msk;
37}

第 34 行：打开 Trace 模块总开关（DEMCR 寄存器的 TRCENA 位），这是访问 DWT 的前提。

第 35 行：将计数器清零。

第 36 行：启动 CYCCNT 计数。之后 DWT->CYCCNT 每个时钟周期自增 1，@ 180 MHz 即每 5.56 ns 加 1。

39static __inline void DWT_DelayFrom(uint32_t start, uint32_t cycles)
40{
41    while ((DWT->CYCCNT - start) < cycles) { ; }
42}

核心延时函数：从 start 时刻开始，等待 cycles 个 CPU 周期。

关键细节：DWT->CYCCNT - start 使用无符号减法，即使 CYCCNT 溢出回绕（0xFFFFFFFF → 0x00000000），减法结果仍然正确。32-bit 计数器在 180 MHz 下约 23.8 秒回绕一次，远大于任何 I2C 延时需求。

为什么用 __inline：这个函数在每个 bit 操作中调用 2 次，内联可以省去函数调用的压栈/跳转开销（~5-8 cycles）。

为什么不用 __NOP() 延时？
__NOP() 循环的延时取决于编译器优化等级（-O2 vs -Og 行为不同），DWT 是硬件计数器，Debug 和 Release 构建的延时完全一致。

3. GPIO 直写宏 — 最快的引脚操作

49/* BSRR[15:0] = set bits, BSRR[31:16] = reset bits */
50#define SCL_HIGH()   (BSP_I2C2_SCL_GPIO_PORT->BSRR = (uint32_t)BSP_I2C2_SCL_PIN)
51#define SCL_LOW()    (BSP_I2C2_SCL_GPIO_PORT->BSRR = (uint32_t)BSP_I2C2_SCL_PIN << 16U)
52#define SCL_READ()   ((BSP_I2C2_SCL_GPIO_PORT->IDR & BSP_I2C2_SCL_PIN) != 0U)
55#define SDA_HIGH()   (BSP_I2C2_SDA_GPIO_PORT->BSRR = (uint32_t)BSP_I2C2_SDA_PIN)
56#define SDA_LOW()    (BSP_I2C2_SDA_GPIO_PORT->BSRR = (uint32_t)BSP_I2C2_SDA_PIN << 16U)
57#define SDA_READ()   ((BSP_I2C2_SDA_GPIO_PORT->IDR & BSP_I2C2_SDA_PIN) != 0U)

BSRR 寄存器（Bit Set/Reset Register）是 STM32 GPIO 的原子操作寄存器：

位域功能示例（Pin_10 = 0x0400）
BSRR[15:0]写 1 → 对应引脚输出高BSRR = 0x0400 → PB10 = 1
BSRR[31:16]写 1 → 对应引脚输出低BSRR = 0x04000000 → PB10 = 0

为什么不用 SPL 函数：GPIO_SetBits() 是函数调用，有压栈、参数传递、跳转开销（~5-8 cycles）。直写 BSRR 编译后变成单条 STR 指令（1-2 cycles），快 4 倍。

开漏模式的妙处：引脚配置为开漏输出后，HIGH = 释放总线（外部 1kΩ 上拉拉高），LOW = 主动拉低。读 IDR 时，即使引脚是输出模式，也能读到总线上的实际电平——如果从设备拉低了 SDA，IDR 读到 0。因此全程不需要切换 GPIO 方向。

4. 延时宏 — SCL 时钟的精确控制

63#define DELAY_HALF()        do { uint32_t _s = DWT->CYCCNT; \
64                                 DWT_DelayFrom(_s, BSP_I2C2_SW_HALF_CYCLE); } while(0)
65
66#define DELAY_HALF_FROM(s)  DWT_DelayFrom((s), BSP_I2C2_SW_HALF_CYCLE)
67
68#define DELAY_LOW_FROM(s)   DWT_DelayFrom((s), \
69                                BSP_I2C2_SW_HALF_CYCLE - BSP_I2C2_SW_SCL_LOW_ADJ)

三个延时宏的用途：

宏延时用途
DELAY_HALF()90 cycles = 500 ns独立等待一个完整半周期（START/STOP 中使用）
DELAY_HALF_FROM(s)从时间点 s 起 90 cyclesSCL 高电平半周期（从 SCL 拉高的那一刻开始计时）
DELAY_LOW_FROM(s)从时间点 s 起 84 cyclesSCL 低电平半周期（扣除 6 cycles GPIO 操作补偿）

为什么 LOW 要减 6 cycles？

在 SCL 低电平期间，CPU 还要做 SDA 设值（1-2 cycles）、读 DWT（1 cycle）、判断/跳转（2-3 cycles）等操作。这些操作本身就占用了 ~6 cycles 的 SCL 低电平时间，如果延时还是满 90 cycles，实际低电平就变成 96 cycles = 533 ns，导致 bit 周期超过 1 μs。

所以 DELAY_LOW_FROM 延时 84 cycles，加上 6 cycles 操作开销，实际低电平 ≈ 90 cycles = 500 ns。

1 MHz I2C 单个 bit 时序： ┌──── 500 ns ────┐┌──── 500 ns ────┐ │ SCL LOW ││ SCL HIGH │ SCL ────┘ └┘ └── │ │ │ SDA ──XX│XXXXXXXXXXXXXXXX │ (数据稳定) │XX │ ↑SDA设值 │ ↑从设备/主机采样 │ │ │ │ │←84cy延时+6cy操作→│←── 90cy 延时 ───→│ │ = 90 cy │ │

5. Clock Stretching — 从设备可以拖慢时钟

76static __inline uint8_t WaitSclHigh(void)
77{
78    uint32_t t0 = DWT->CYCCNT;
79    while (!SCL_READ())
80    {
81        if ((DWT->CYCCNT - t0) > BSP_I2C2_SW_STRETCH_TIMEOUT) { return 0U; }
82    }
83    return 1U;
84}

Clock Stretching：I2C 协议允许从设备在 SCL 高电平时把 SCL 拉低，告诉主机"我还没准备好"。主机释放 SCL 后必须检查 SCL 是否真的变高了。

第 79 行：循环读 SCL，等它变高。如果从设备没有 stretch，SCL 已经是高的，循环立即退出（0 次迭代）。

第 81 行：超时保护（1,800,000 cycles = 10 ms @ 180 MHz）。如果从设备卡死不释放 SCL，10 ms 后返回 0 表示失败。

正常情况：你的电机 IC 支持 1 MHz 且不做 clock stretching，所以这个函数实际上每次都是"读一次 SCL = 高，立即返回 1"。

6. Bit 级原语 — 发送和接收单个 bit

6.1 SendBit — 发送一个 bit

90static __inline void SendBit(uint8_t bit)
91{
92    uint32_t t;
93
94    /* SCL low phase: set SDA then wait */
95    if (bit) { SDA_HIGH(); } else { SDA_LOW(); }
96    t = DWT->CYCCNT;
97    DELAY_LOW_FROM(t);
98
99    /* SCL high phase */
100    SCL_HIGH();
101    t = DWT->CYCCNT;
102    (void)WaitSclHigh();
103    DELAY_HALF_FROM(t);
104
105    SCL_LOW();
106}

逐行分析：

第 95 行：在 SCL 低电平期间设置 SDA 数据线。I2C 规定 SDA 只能在 SCL 为低时变化。

第 96-97 行：记录当前时刻，等待 84 cycles（补偿后的低电平时间）。加上第 95 行的操作开销，SCL 实际低电平 ≈ 90 cycles = 500 ns。

第 100 行：释放 SCL（开漏，外部上拉拉高）。

第 101 行：立即记录 SCL 拉高的时刻，作为高电平计时起点。

第 102 行：检查 SCL 是否真的变高了（clock stretching 检测）。

第 103 行：从第 101 行记录的时刻起，等待 90 cycles = 500 ns 的 SCL 高电平时间。从设备在此期间采样 SDA。

第 105 行：拉低 SCL，结束这个 bit。

SendBit(1) 时序： SDA ──────────────────────────────────────────── 已经是高(释放状态) 数据稳定,从设备采样 SCL ──┐ 84cy + 操作 ┌──── 90 cycles ────┐ │ ≈ 500 ns │ 500 ns │ └──────────────────┘ └── ↑ ↑ ↑ 上一bit结束 SCL_HIGH() SCL_LOW() SendBit(0) 时序： SDA ──┐ ┌── │ SDA 拉低 │ └───────────────────────────────────────┘ SCL ──┐ ┌────────────────────┐ └──────────────────┘ └──

6.2 RecvBit — 接收一个 bit

108static __inline uint8_t RecvBit(void)
109{
110    uint8_t  bit;
111    uint32_t t;
112
113    /* Release SDA so slave can drive it */
114    SDA_HIGH();
115    t = DWT->CYCCNT;
116    DELAY_LOW_FROM(t);
117
118    /* SCL high phase: sample SDA */
119    SCL_HIGH();
120    t = DWT->CYCCNT;
121    if (!WaitSclHigh()) { return 1U; }  /* timeout → NACK */
122    bit = SDA_READ() ? 1U : 0U;
123    DELAY_HALF_FROM(t);
124
125    SCL_LOW();
126    return bit;
127}

第 114 行：主机释放 SDA（写高 = 开漏断开），让从设备控制 SDA 电平。

第 115-116 行：SCL 低电平等待，给从设备时间准备数据。

第 119 行：释放 SCL 变高。

第 121 行：如果 SCL 超时未变高，返回 1（等效 NACK），上层会当作通信失败处理。

第 122 行：在 SCL 高电平期间采样 SDA — 这就是从设备发送的数据 bit。

开漏模式为什么能读：主机写 HIGH 只是断开了自己的下拉 MOSFET，SDA 线的电平由外部上拉 + 从设备共同决定。如果从设备拉低 SDA，IDR 读到 0；如果从设备也释放，上拉电阻拉到高，IDR 读到 1。

7. Byte 级原语 — 发送和接收字节

7.1 SendByte — 发送 1 字节 + 读 ACK

133/** Send 8 bits MSB-first, read ACK. Returns 1 if ACK received, 0 if NACK. */
134static uint8_t SendByte(uint8_t byte)
135{
136    uint8_t i;
137    for (i = 0U; i < 8U; i++)
138    {
139        SendBit((byte & 0x80U) ? 1U : 0U);
140        byte <<= 1U;
141    }
142    /* ACK bit: slave pulls SDA low = ACK */
143    return (RecvBit() == 0U) ? 1U : 0U;
144}

第 139 行：检查最高位（bit7），I2C 是 MSB-first（最高位先发）。byte & 0x80 非零 → 发 1，否则发 0。

第 140 行：左移一位，下次循环发次高位。

第 143 行：8 个数据 bit 发完后，第 9 个时钟周期是 ACK。主机释放 SDA（RecvBit 内部会做），从设备拉低 SDA 表示 ACK（确认收到），保持高电平表示 NACK（未应答）。

返回值：1 = ACK（通信正常），0 = NACK（设备未响应或地址错误）。

SendByte(0xA5) = 10100101 的时序： bit7 bit6 bit5 bit4 bit3 bit2 bit1 bit0 ACK SDA ────┐ ┌──┐ ┐ ┌──┐ ┐ ┌──╮ ╭── (1) │(0) │ │(1) (0) │(0) │ │(1) (0) │(1) │ ╰────╯ └────┘ └─────────┘────┘ └─────────┘────┘ 从设备拉低 = ACK SCL ┌┐ ┌┐ ┌┐ ┌┐ ┌┐ ┌┐ ┌┐ ┌┐ ┌┐ └┘ └┘ └┘ └┘ └┘ └┘ └┘ └┘ └┘ ←──────── 8 data bits ──────────→ ←ACK→ 共 9 个时钟

7.2 RecvByte — 接收 1 字节 + 发 ACK/NACK

146/** Receive 8 bits MSB-first, send ACK (ack=1) or NACK (ack=0). */
147static uint8_t RecvByte(uint8_t ack)
148{
149    uint8_t byte = 0U;
150    uint8_t i;
151    for (i = 0U; i < 8U; i++)
152    {
153        byte = (uint8_t)((byte << 1U) | RecvBit());
154    }
155    /* Master sends ACK (SDA low) or NACK (SDA high) */
156    SendBit(ack ? 0U : 1U);
157    return byte;
158}

第 153 行：每次循环左移 byte，然后 OR 上从 SDA 读到的 bit。8 次循环后得到完整字节。

第 156 行：第 9 个时钟，主机发 ACK 或 NACK。注意逻辑反转：ack=1 时发送 bit 0（SDA 低 = ACK），ack=0 时发送 bit 1（SDA 高 = NACK）。

何时发 ACK / NACK：读取多字节时，中间字节发 ACK（"继续发"），最后一个字节发 NACK（"不要了，准备 STOP"）。

8. I2C 协议条件 — START / STOP / Repeated START

8.1 START 条件

164static void I2C_Start(void)
165{
166    SDA_HIGH();
167    SCL_HIGH();
168    DELAY_HALF();
169    (void)WaitSclHigh();
170    SDA_LOW();          /* SDA falls while SCL high = START */
171    DELAY_HALF();
172    SCL_LOW();
173}

I2C START 条件的定义：SCL 为高时，SDA 从高跳变为低。

第 166-167 行：先确保两条线都是高电平（空闲状态）。

第 168-169 行：等待半周期，确保总线稳定 + SCL 确实为高。

第 170 行：在 SCL 高电平期间拉低 SDA — 这就是 START。

第 171 行：保持半周期的 setup time。

第 172 行：拉低 SCL，准备发第一个 bit。

START 条件： SDA ────────────┐ │ SDA 在 SCL 高时下降 └────────────────────── SCL ────────────────────┐ │ 然后 SCL 拉低 └────────────── 空闲 ←setup→ 第一个 bit 开始

8.2 STOP 条件

175static void I2C_Stop(void)
176{
177    uint32_t t;
178
179    SDA_LOW();
180    t = DWT->CYCCNT;
181    DELAY_LOW_FROM(t);
182
183    SCL_HIGH();
184    t = DWT->CYCCNT;
185    (void)WaitSclHigh();
186    DELAY_HALF_FROM(t);
187
188    SDA_HIGH();         /* SDA rises while SCL high = STOP */
189    DELAY_HALF();
190}

I2C STOP 条件的定义：SCL 为高时，SDA 从低跳变为高。

第 179 行：先确保 SDA 为低（为后面的上升沿做准备）。

第 183-186 行：释放 SCL 变高，等待半周期。

第 188 行：在 SCL 高电平期间释放 SDA — SDA 上升沿 = STOP。

第 189 行：等待半周期，确保 STOP 被从设备识别，总线恢复空闲。

STOP 条件： SDA ──────────────────────────┌──────── 确保低 │ SDA 在 SCL 高时上升 ─────────────────────────┘ SCL ──────┌─────────────────────────── │ SCL 先拉高 ──────┘ 最后bit ← setup → 空闲

8.3 Repeated START

192static void I2C_RepeatedStart(void)
193{
194    SDA_HIGH();
195    DELAY_HALF();
196    SCL_HIGH();
197    DELAY_HALF();
198    (void)WaitSclHigh();
199    SDA_LOW();          /* SDA falls while SCL high = repeated START */
200    DELAY_HALF();
201    SCL_LOW();
202}

Repeated START 用于在不释放总线（不发 STOP）的情况下切换传输方向。典型场景：写完寄存器地址后，不 STOP，直接 Repeated START 切换为读模式。

第 194 行：先释放 SDA 变高（上一个操作可能留下 SDA 低）。

第 196 行：然后释放 SCL 变高。此时 SDA=高，SCL=高，与 START 前的空闲状态相同。

第 199 行：在 SCL 高时拉低 SDA — 再次产生 START 条件。

与 I2C_Start 的区别：不需要检查总线是否空闲（因为总线已经是自己控制的）。

9. 公开 API — Init 初始化

208ErrorStatus BSP_I2C2_Init(void)
209{
210    GPIO_InitTypeDef gpioInit;
211
212    /* 1. Enable GPIO clock */
213    RCC_AHB1PeriphClockCmd(BSP_I2C2_SCL_GPIO_CLK | BSP_I2C2_SDA_GPIO_CLK,
214                           ENABLE);
215
216    /* 2. Configure SCL & SDA as output open-drain, no pull, 50 MHz */
217    gpioInit.GPIO_Mode  = GPIO_Mode_OUT;
218    gpioInit.GPIO_Speed = GPIO_Speed_50MHz;
219    gpioInit.GPIO_OType = GPIO_OType_OD;
220    gpioInit.GPIO_PuPd  = GPIO_PuPd_NOPULL;
221
222    gpioInit.GPIO_Pin = BSP_I2C2_SCL_PIN;
223    GPIO_Init(BSP_I2C2_SCL_GPIO_PORT, &gpioInit);
224
225    gpioInit.GPIO_Pin = BSP_I2C2_SDA_PIN;
226    GPIO_Init(BSP_I2C2_SDA_GPIO_PORT, &gpioInit);
227
228    /* 3. Release both lines (idle high) */
229    SCL_HIGH();
230    SDA_HIGH();
231
232    /* 4. Enable DWT cycle counter */
233    DWT_Init();
234
235    /* 5. Check bus stuck (SDA held low) and recover */
236    if (!SDA_READ())
237    {
238        BSP_I2C2_RecoverBus();
239        if (!SDA_READ()) { return ERROR; }
240    }
241
242    return SUCCESS;
243}

与硬件 I2C 初始化的关键区别：

硬件 I2C（旧）软件 I2C（新）
GPIO 模式GPIO_Mode_AF（复用功能）GPIO_Mode_OUT（普通输出）
AF 配置需要 GPIO_PinAFConfig()不需要，引脚不连接到 I2C 外设
I2C 外设使能 RCC_APB1、I2C_Init()不使用，不使能
时钟源APB1 45 MHz → I2C 外设分频DWT->CYCCNT 180 MHz

第 217 行 GPIO_Mode_OUT：这是关键——引脚是普通 GPIO 输出，不连接到 I2C2 硬件外设。

第 219 行 GPIO_OType_OD：开漏输出，I2C 总线必须的"线与"特性。

第 220 行 GPIO_PuPd_NOPULL：不启用内部上拉，使用外部 1 kΩ 上拉。

第 236-240 行：上电时检查 SDA 是否被从设备拉低（总线卡死），如果是则尝试恢复。恢复后仍然卡死则返回 ERROR。

10. 公开 API — WriteReg 写寄存器

247ErrorStatus BSP_I2C2_WriteReg(uint8_t devAddr, uint16_t reg,
248                               const uint8_t *pData, uint16_t len)
249{
250    uint16_t i;
251
252    __disable_irq();
253
254    I2C_Start();
255
256    /* Address + W */
257    if (!SendByte((uint8_t)(devAddr << 1U)))        { goto error; }
258
259    /* 16-bit register address, big-endian */
260    if (!SendByte((uint8_t)(reg >> 8U)))             { goto error; }
261    if (!SendByte((uint8_t)(reg & 0xFFU)))           { goto error; }
262
263    /* Data bytes */
264    for (i = 0U; i < len; i++)
265    {
266        if (!SendByte(pData[i]))                     { goto error; }
267    }
268
269    I2C_Stop();
270    __enable_irq();
271    return SUCCESS;
272
273error:
274    I2C_Stop();
275    __enable_irq();
276    BSP_I2C2_RecoverBus();
277    return ERROR;
278}

第 252 行 __disable_irq()：关闭全局中断，防止中断打断 bit-bang 时序。

第 257 行：发送设备地址。I2C 地址格式：[7-bit地址][R/W]，devAddr << 1 左移一位，最低位 0 = 写。

第 260-261 行：发送 16-bit 寄存器地址，先高字节后低字节（大端）。

第 264-267 行：发送数据字节。典型场景 len=2（16-bit 寄存器值）。

任何一步 NACK → goto error：发 STOP 释放总线 → 开中断 → 尝试恢复总线 → 返回 ERROR。

关键：error 路径也会 __enable_irq()，确保不会永久关中断。

11. 公开 API — ReadReg 读寄存器

282ErrorStatus BSP_I2C2_ReadReg(uint8_t devAddr, uint16_t reg,
283                              uint8_t *pData, uint16_t len)
284{
285    uint16_t i;
286
287    if (len == 0U) { return SUCCESS; }
288
289    __disable_irq();
290
291    /* Write phase: START + addr+W + 16-bit reg addr */
292    I2C_Start();
293    if (!SendByte((uint8_t)(devAddr << 1U)))         { goto error; }
294    if (!SendByte((uint8_t)(reg >> 8U)))              { goto error; }
295    if (!SendByte((uint8_t)(reg & 0xFFU)))            { goto error; }
296
297    /* Read phase: repeated START + addr+R */
298    I2C_RepeatedStart();
299    if (!SendByte((uint8_t)((devAddr << 1U) | 0x01U))) { goto error; }
300
301    /* Read data bytes: ACK all except last (NACK) */
302    for (i = 0U; i < len; i++)
303    {
304        pData[i] = RecvByte((i < (len - 1U)) ? 1U : 0U);
305    }
306
307    I2C_Stop();
308    __enable_irq();
309    return SUCCESS;
310
311error:
312    I2C_Stop();
313    __enable_irq();
314    BSP_I2C2_RecoverBus();
315    return ERROR;
316}

读操作分两阶段：

Write Phase（第 292-295 行）：先告诉从设备"我要读哪个寄存器"。发 START → 设备地址+W → 16-bit 寄存器地址。

Read Phase（第 298-305 行）：切换方向读数据。发 Repeated START → 设备地址+R → 接收数据字节。

第 299 行：devAddr << 1 | 0x01 — 最低位 1 = 读。

第 304 行：i < (len-1) → 中间字节发 ACK（1），最后一个字节发 NACK（0），通知从设备停止发送。

12. 公开 API — Scan 总线扫描

328ErrorStatus BSP_I2C2_Scan(uint8_t *pAddrList, uint8_t *pCount)
329{
330    uint8_t addr;
331    *pCount = 0U;
332
333    for (addr = 1U; addr <= 126U; addr++)
334    {
335        __disable_irq();
336        I2C_Start();
337        if (SendByte((uint8_t)(addr << 1U)))
338        {
339            pAddrList[(*pCount)++] = addr;
340        }
341        I2C_Stop();
342        __enable_irq();
343    }
344
345    return SUCCESS;
346}

扫描 I2C 总线上所有设备。原理：依次尝试向地址 1~126 发 START + 地址+W，如果收到 ACK 说明该地址有设备。

中断保护策略：每个地址单独 disable/enable，每次关中断仅 ~10 μs（START + 9 bit + STOP），对 UART 等中断几乎无影响。

总耗时：126 个地址 × ~10 μs ≈ 1.3 ms（远快于硬件 I2C 版本的 400 kHz 扫描）。

13. 公开 API — RecoverBus 总线恢复

350void BSP_I2C2_RecoverBus(void)
351{
352    uint8_t  i;
353    uint32_t t;
354
355    /* Toggle SCL 9 times to clock out stuck slave */
356    for (i = 0U; i < BSP_I2C2_SW_RECOVER_PULSES; i++)
357    {
358        SCL_LOW();
359        t = DWT->CYCCNT;
360        DELAY_HALF_FROM(t);
361        SCL_HIGH();
362        t = DWT->CYCCNT;
363        DELAY_HALF_FROM(t);
364    }
365
366    /* Manual STOP condition */
367    SDA_LOW();
368    t = DWT->CYCCNT;
369    DELAY_HALF_FROM(t);
370    SCL_HIGH();
371    t = DWT->CYCCNT;
372    DELAY_HALF_FROM(t);
373    SDA_HIGH();
374    DELAY_HALF();
375
376    /* Set recovery flag if bus is now free */
377    if (SDA_READ())
378    {
379        s_busRecovered = 1U;
380    }
381}

为什么需要总线恢复？

如果上一次 I2C 通信中途被打断（比如主机复位），从设备可能停留在"正在发送数据"的状态，持续拉低 SDA。此时总线卡死，无法发 START。

恢复原理：

1. 第 356-364 行：发送 9 个 SCL 脉冲。从设备看到 SCL 时钟后会继续移位输出残余数据，9 个脉冲足够让任何字节传输完成（最坏情况 8 data + 1 ACK）。

2. 第 367-374 行：发送一个手动 STOP 条件（SDA 从低到高，SCL 为高），通知从设备释放总线。

3. 第 377-380 行：检查 SDA 是否已经恢复高电平（空闲），如果是则设置恢复标志。

14. 中断保护策略总结

函数关中断范围最大关中断时间 @ 1 MHz
WriteReg（写1寄存器）START → STOP~47 μs
ReadReg（读1寄存器）START → STOP~67 μs
Scan（单个地址）START → STOP~10 μs
RecoverBus不关中断—
典型场景（写1+读1）：WriteReg 47 μs + ReadReg 67 μs = 114 μs，中间有短暂的开中断窗口。
UART @ 115200 bps 一个字节 87 μs，67 μs 关中断期间最多积压不到 1 个字节，硬件 DR 寄存器可以缓冲，不会丢数据。
