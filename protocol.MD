# 串口通讯协议文档

## 1. 概述

本协议定义了PC上位机与STM32开发板之间的串口通讯规范。STM32通过I2C与电机驱动IC进行交互，PC通过本协议对其进行控制与数据读取。

---

## 2. 串口参数

| 参数 | 值 |
|------|------|
| 波特率 | 默认 115200，可配置（见4.2节 `0x03` 命令） |
| 数据位 | 8 |
| 停止位 | 1 |
| 校验位 | 无 |
| 流控 | 无 |

---

## 3. 帧格式

### 3.1 控制帧

用于PC下发指令、STM32返回响应，低频、可靠性优先。

```
+--------+--------+----------+----------+----------+--------+----------+
| 0xAA   | 0x55   | 帧序号   | 命令码   | 数据长度 | 数据   | CRC16    |
| 1 字节 | 1 字节 | 1 字节   | 1 字节   | 1 字节   | n 字节 | 2 字节   |
+--------+--------+----------+----------+----------+--------+----------+
```

| 字段 | 长度 | 说明 |
|------|------|------|
| 帧头 | 2 字节 | 固定为 `0xAA 0x55` |
| 帧序号 | 1 字节 | 发送方递增计数，0~255 循环，响应帧回传相同序号 |
| 命令码 | 1 字节 | 标识指令类型，见第4节 |
| 数据长度 | 1 字节 | 数据字段的字节数，范围 0~255 |
| 数据 | n 字节 | 指令参数或返回内容，由命令码决定 |
| CRC16 | 2 字节 | 对帧序号、命令码、数据长度、数据计算，高字节在前 |

> CRC16 校验范围：帧序号 + 命令码 + 数据长度 + 数据（不含帧头）

---

### 3.2 数据流帧

用于STM32向PC高频上报示波器采样数据，实时性优先，开销最小。

```
+--------+------------+--------+--------+----------+
| 0xBB   | 通道掩码   | LEN    | 数据   | XOR校验  |
| 1 字节 | 1 字节     | 1 字节 | n 字节 | 1 字节   |
+--------+------------+--------+--------+----------+
```

| 字段 | 长度 | 说明 |
|------|------|------|
| 帧头 | 1 字节 | 固定为 `0xBB` |
| 通道掩码 | 1 字节 | 每个bit对应一个通道，bit=1表示该通道有效 |
| LEN | 1 字节 | 数据段字节数，= 通道掩码中bit为1的数量 × 2 |
| 数据 | n 字节 | 各通道采样值，按通道编号从低到高排列，每通道2字节（int16，大端，高字节在前） |
| XOR校验 | 1 字节 | 对通道掩码、LEN和数据所有字节逐字节异或 |

> **I2C 读取失败处理**：若某通道在采样时 I2C 读取失败，该通道数据填 `0x0000`，数据流继续上报，不中断采样。上位机若需区分寄存器值为零和读取失败，应结合应用层逻辑判断。

**通道掩码示例：**

| 掩码值 | 含义 |
|--------|------|
| `0x01` | 仅通道0 |
| `0x03` | 通道0、通道1 |
| `0x0F` | 通道0~3 |

---

### 3.3 帧类型识别

接收端通过首字节判断帧类型：

| 首字节 | 帧类型 |
|--------|--------|
| `0xAA` | 控制帧（需继续读取第二字节确认 `0x55`） |
| `0xBB` | 数据流帧 |

---

## 4. 命令码定义

### 4.1 分组规划

| 范围 | 类别 |
|------|------|
| `0x00` | 心跳 |
| `0x01 ~ 0x1F` | 系统控制类 |
| `0x20 ~ 0x4F` | 寄存器读写类 |
| `0x50 ~ 0x7F` | 示波器控制类 |
| `0x80 ~ 0xBF` | 预留扩展 |
| `0xC0 ~ 0xFE` | 用户自定义 |
| `0xFF` | 保留 |

---

### 4.2 系统控制组（0x00~0x1F）

#### `0x00` 心跳

| 项目 | 内容 |
|------|------|
| 方向 | 双向 |
| 数据段 | 空（LEN=0） |
| 响应 | STM32 echo 原帧 |

---

#### `0x01` 错误响应

| 项目 | 内容 |
|------|------|
| 方向 | STM32→PC |
| 数据段 | 1 字节错误码（见4.5节） |
| 说明 | STM32遇到异常时主动回复 |

---

#### `0x02` 设置电机 IC 地址

| 项目 | 内容 |
|------|------|
| 方向 | PC→STM32 |
| 数据段 | 1 字节，7-bit I2C 地址 |
| 成功响应 | 同 CMD+SEQ，LEN=0（地址已更新） |
| 失败条件 | 地址为 `0x00` 或 > `0x7F`；I2C 无应答 |
| 失败响应 | 错误响应帧，错误码 `0x03` |
| 备注 | 失败时 `g_motorIcAddr` 保持原值不变 |

---

#### `0x03` 设置波特率

| 项目 | 内容 |
|------|------|
| 方向 | PC→STM32 |
| 数据段 | 1 字节，波特率索引（0x00~0x07） |
| 成功响应 | 用**旧波特率**发送同 CMD+SEQ、LEN=0 的响应帧，发送完成后切换到新波特率 |
| 失败条件 | 索引 > `0x07` |
| 失败响应 | 错误响应帧，错误码 `0x03` |

波特率索引表：

| 索引 | 波特率 |
|------|--------|
| `0x00` | 9600 |
| `0x01` | 19200 |
| `0x02` | 38400 |
| `0x03` | 57600 |
| `0x04` | 115200（默认） |
| `0x05` | 230400 |
| `0x06` | 460800 |
| `0x07` | 921600 |

---

#### `0x04` 系统复位

| 项目 | 内容 |
|------|------|
| 方向 | PC→STM32 |
| 数据段 | 空（LEN=0） |
| 成功响应 | 同 CMD+SEQ、LEN=0 的响应帧，发送完成后执行软件复位 |
| 复位后状态 | 波特率恢复默认（115200），`g_motorIcAddr` 清零 |
| 约定 | **上位机发复位前必须先发 `0x03` 将波特率切回 115200** |

---

#### `0x05` 电机应答测试

| 项目 | 内容 |
|------|------|
| 方向 | PC→STM32 |
| 数据段 | 空（LEN=0） |
| 成功响应 | 同 CMD+SEQ，LEN=0 |
| 失败条件 | `g_motorIcAddr` 为 `0x00`；I2C 无应答 |
| 失败响应 | 错误响应帧，错误码 `0x03` |

---

#### `0x06` 调试信息

| 项目 | 内容 |
|------|------|
| 方向 | STM32→PC |
| 数据段 | ASCII 字符串，最长 255 字节 |
| 触发时机 | STM32 主动发送，用于说明错误详情（如启动采样失败原因） |

---

#### `0x07` I2C 总线扫描

| 项目 | 内容 |
|------|------|
| 方向 | PC→STM32 |
| 数据段 | 1 字节，总线索引（`0x01`=I2C1 / `0x02`=I2C2 / `0x03`=I2C3） |
| 成功响应 | 数据段：`[count][addr0][addr1]...[addrN-1]`，count 为发现的设备数量，addr 为 7-bit 设备地址 |
| 失败条件 | 总线索引不在 1~3 范围内；总线扫描操作本身失败 |
| 失败响应 | 错误响应帧，错误码 `0x03` |
| 备注 | 扫描地址范围 0x01~0x7E，逐地址发送 START+地址+STOP 检测 ACK |

---

### 4.3 寄存器读写组（0x20~0x4F）

#### `0x20` 读单个寄存器

| 项目 | 内容 |
|------|------|
| 方向 | PC→STM32 |
| 数据段 | 2 字节：`[寄存器地址高字节][寄存器地址低字节]`，大端 |
| 成功响应 | 返回帧，数据段 2 字节，大端（高字节在前） |
| 失败响应 | 错误响应帧，错误码 `0x03` |

---

#### `0x21` 写单个寄存器

| 项目 | 内容 |
|------|------|
| 方向 | PC→STM32 |
| 数据段 | 4 字节：`[寄存器地址高字节][寄存器地址低字节][数据高字节][数据低字节]`，大端 |
| 成功响应 | 同 CMD+SEQ，LEN=0 |
| 失败响应 | 错误响应帧，错误码 `0x03` |

---

#### `0x22` 批量读寄存器

| 项目 | 内容 |
|------|------|
| 方向 | PC→STM32 |
| 数据段 | 4 字节：`[起始寄存器地址高字节][起始寄存器地址低字节][读取数量高字节][读取数量低字节]`，大端；读取数量范围 1~32130 |
| 成功响应 | 多帧返回，每帧数据段：`[包序号][总包数][数据...]`，每包最多 252 字节（126个寄存器），每个寄存器 2 字节大端 |
| 失败响应 | 错误响应帧，错误码 `0x03` |
| 备注 | 分包发送过程中收到的所有控制帧全部忽略；采样进行中收到此命令直接返回错误码 `0x03` |

**地址步长**：起始寄存器地址为字节地址，每个后续寄存器地址递增 **+2**。例如起始地址 `0x0010`，读取 3 个寄存器，依次访问 `0x0010`、`0x0012`、`0x0014`。

**包序号**：从 **0** 开始计数，第一包序号为 `0x00`，最后一包序号为 `总包数 - 1`。

---

### 4.4 示波器控制组（0x50~0x7F）

#### 采样默认值

| 参数 | 默认值 |
|------|--------|
| 采样间隔 | 索引 `0x05`（1000 us） |
| 通道掩码 | `0x01`（仅通道0） |
| 通道寄存器映射 | 全部 `0xFFFF`（未映射） |

---

#### `0x50` 启动采样

| 项目 | 内容 |
|------|------|
| 方向 | PC→STM32 |
| 数据段 | 空（LEN=0） |
| 成功响应 | 同 CMD+SEQ，LEN=0；之后持续上报数据流帧 |
| 失败条件 | 所有通道寄存器映射均为 `0xFFFF` |
| 失败响应 | 错误响应帧，错误码 `0x03`；同时发 `0x06` 调试信息说明原因 |

---

#### `0x51` 停止采样

| 项目 | 内容 |
|------|------|
| 方向 | PC→STM32 |
| 数据段 | 空（LEN=0） |
| 成功响应 | 同 CMD+SEQ，LEN=0 |
| 说明 | 立即停止 TIM6，停止数据流上报；**通道配置（掩码、寄存器映射）保持不变**，下次 `0x50` 可直接重启采样 |

---

#### `0x52` 设置采样间隔

| 项目 | 内容 |
|------|------|
| 方向 | PC→STM32 |
| 数据段 | 1 字节，采样间隔索引（0x00~0x07） |
| 成功响应 | 同 CMD+SEQ，LEN=0 |
| 失败条件 | 索引 > `0x07` |
| 失败响应 | 错误响应帧，错误码 `0x03` |

采样间隔索引表：

| 索引 | 采样间隔 | 对应频率 |
|------|----------|----------|
| `0x00` | 100 us | 10000 Hz |
| `0x01` | 200 us | 5000 Hz |
| `0x02` | 300 us | 3333 Hz |
| `0x03` | 500 us | 2000 Hz |
| `0x04` | 750 us | 1333 Hz |
| `0x05` | 1000 us | 1000 Hz（默认） |
| `0x06` | 1500 us | 667 Hz |
| `0x07` | 2000 us | 500 Hz |

---

#### `0x53` 设置采样通道

| 项目 | 内容 |
|------|------|
| 方向 | PC→STM32 |
| 数据段 | 1 字节，通道掩码（bit0~bit7 对应通道 0~7） |
| 成功响应 | 同 CMD+SEQ，LEN=0 |
| 失败条件 | 掩码为 `0x00` |
| 失败响应 | 错误响应帧，错误码 `0x03` |

---

#### `0x54` 设置通道寄存器映射

| 项目 | 内容 |
|------|------|
| 方向 | PC→STM32 |
| 数据段 | 16 字节，每通道 2 字节寄存器地址，大端，共 8 个通道按顺序排列 |
| 成功响应 | 同 CMD+SEQ，LEN=0 |
| 不使用的通道 | 填 `0xFFFF` |

---

### 4.5 错误码定义

| 错误码 | 名称 | 说明 |
|--------|------|------|
| `0x01` | CRC校验失败 | 收到的控制帧CRC校验不通过，错误响应帧序号固定填 `0xFF` |
| `0x02` | 未知命令码 | 命令码未定义或不支持 |
| `0x03` | 执行失败 | 命令执行过程中发生错误（如I2C通讯失败） |

---

## 5. 心跳机制

- PC端每隔 **1秒** 发送一次心跳控制帧（命令码 `0x00`，数据长度 `0x00`）
- STM32收到后立即回复相同格式的心跳帧
- PC端若连续 **3次** 未收到心跳回复，判定连接断开，界面给出断线提示

---

## 6. 超时与重发机制

- 本机制仅适用于普通控制帧，**心跳帧独立处理，不走本机制**
- PC发出控制帧后，等待STM32响应超时时间为 **100ms**
- 超时后自动重发，最多重发 **2次**（共发送3次）
- 3次均无响应则判定本次指令失败，上层报错
- 重发时帧序号保持不变，STM32收到重复帧序号时重新执行并回复（不做去重处理）

---

## 7. CRC16 计算说明

- 算法：CRC16-MODBUS（多项式 `0x8005`，初始值 `0xFFFF`，低位先行）
- 校验范围：帧序号 + 命令码 + 数据长度 + 数据
- 结果2字节，**高字节在前（大端）** 存入帧
- 注意：标准 MODBUS 协议中 CRC16 结果为低字节在前（小端），本协议规定为大端，实现时若使用现成库需手动交换两字节后再写入帧
