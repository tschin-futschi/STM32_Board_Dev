# 串口通讯协议文档

## 1. 概述

本协议定义了PC上位机与STM32开发板之间的串口通讯规范。STM32通过I2C与摄像头模组IC进行交互，PC通过本协议对其进行控制与数据读取。

---

## 2. 串口参数

| 参数 | 值 |
|------|------|
| 波特率 | 默认 115200，可配置（9600 / 19200 / 38400 / 57600 / 115200 / 230400 / 460800 / 921600） |
| 数据位 | 8 |
| 停止位 | 1 |
| 校验位 | 无 |
| 流控 | 无 |

---

## 3. 帧格式

### 3.1 控制帧

用于PC下发指令、STM32返回响应，低频、可靠性优先。

```
+--------+--------+----------+----------+----------+--------+----------+
| 0xAA   | 0x55   | 帧序号   | 命令码   | 数据长度 | 数据   | CRC16    |
| 1 字节 | 1 字节 | 1 字节   | 1 字节   | 1 字节   | n 字节 | 2 字节   |
+--------+--------+----------+----------+----------+--------+----------+
```

| 字段 | 长度 | 说明 |
|------|------|------|
| 帧头 | 2 字节 | 固定为 `0xAA 0x55` |
| 帧序号 | 1 字节 | 发送方递增计数，0~255 循环，响应帧回传相同序号 |
| 命令码 | 1 字节 | 标识指令类型，见第4节 |
| 数据长度 | 1 字节 | 数据字段的字节数，范围 0~255 |
| 数据 | n 字节 | 指令参数或返回内容，由命令码决定 |
| CRC16 | 2 字节 | 对帧序号、命令码、数据长度、数据计算，高字节在前 |

> CRC16 校验范围：帧序号 + 命令码 + 数据长度 + 数据（不含帧头）

---

### 3.2 数据流帧

用于STM32向PC高频上报示波器采样数据，实时性优先，开销最小。

```
+--------+------------+--------+----------+
| 0xBB   | 通道掩码   | 数据   | XOR校验  |
| 1 字节 | 1 字节     | n 字节 | 1 字节   |
+--------+------------+--------+----------+
```

| 字段 | 长度 | 说明 |
|------|------|------|
| 帧头 | 1 字节 | 固定为 `0xBB` |
| 通道掩码 | 1 字节 | 每个bit对应一个通道，bit=1表示该通道有效 |
| 数据 | n 字节 | 各通道采样值，按通道编号从低到高排列，每通道2字节（int16，大端，高字节在前） |
| XOR校验 | 1 字节 | 对通道掩码和数据所有字节逐字节异或 |

**通道掩码示例：**

| 掩码值 | 含义 |
|--------|------|
| `0x01` | 仅通道0 |
| `0x03` | 通道0、通道1 |
| `0x0F` | 通道0~3 |

---

### 3.3 帧类型识别

接收端通过首字节判断帧类型：

| 首字节 | 帧类型 |
|--------|--------|
| `0xAA` | 控制帧（需继续读取第二字节确认 `0x55`） |
| `0xBB` | 数据流帧 |

---

## 4. 命令码定义

### 4.1 分组规划

| 范围 | 类别 |
|------|------|
| `0x00` | 心跳 |
| `0x01 ~ 0x1F` | 系统控制类（复位、握手、状态查询等） |
| `0x20 ~ 0x4F` | 寄存器读写类 |
| `0x50 ~ 0x7F` | 示波器控制类（开始/停止采集、配置通道等） |
| `0x80 ~ 0xBF` | 预留扩展 |
| `0xC0 ~ 0xFE` | 用户自定义 |
| `0xFF` | 保留 |

### 4.2 已定义命令码

| 命令码 | 名称 | 方向 | 说明 |
|--------|------|------|------|
| `0x00` | 心跳 | 双向 | PC定期发送，STM32以相同命令码回复，数据段为空 |
| `0x01` | 错误响应 | STM32→PC | STM32遇到异常时主动回复，数据段为1字节错误码，见4.3节 |

> 其余命令码按需在对应分组内扩展定义。

### 4.3 错误码定义

| 错误码 | 名称 | 说明 |
|--------|------|------|
| `0x01` | CRC校验失败 | 收到的控制帧CRC校验不通过，此时帧序号不可信，错误响应帧序号固定填 `0xFF` |
| `0x02` | 未知命令码 | 命令码未定义或不支持 |
| `0x03` | 执行失败 | 命令执行过程中发生错误（如I2C通讯失败） |

---

## 5. 心跳机制

- PC端每隔 **1秒** 发送一次心跳控制帧（命令码 `0x00`，数据长度 `0x00`）
- STM32收到后立即回复相同格式的心跳帧
- PC端若连续 **3次** 未收到心跳回复，判定连接断开，界面给出断线提示

---

## 6. 超时与重发机制

- 本机制仅适用于普通控制帧，**心跳帧独立处理，不走本机制**
- PC发出控制帧后，等待STM32响应超时时间为 **100ms**
- 超时后自动重发，最多重发 **2次**（共发送3次）
- 3次均无响应则判定本次指令失败，上层报错
- 重发时帧序号保持不变（与原始帧相同），以便STM32识别重复帧
- STM32收到重复帧序号时，应重新执行并回复（不做去重处理）

---

## 7. CRC16 计算说明

- 算法：CRC16-MODBUS（多项式 `0x8005`，初始值 `0xFFFF`，低位先行）
- 校验范围：帧序号 + 命令码 + 数据长度 + 数据
- 结果2字节，**高字节在前（大端）** 存入帧
- 注意：标准 MODBUS 协议中 CRC16 结果为低字节在前（小端），本协议规定为大端，实现时若使用现成库需手动交换两字节后再写入帧
