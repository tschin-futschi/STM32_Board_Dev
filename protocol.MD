# 串口通讯协议文档

## 1. 概述

本协议定义了PC上位机与STM32开发板之间的串口通讯规范。STM32通过I2C与电机驱动IC进行交互，PC通过本协议对其进行控制与数据读取。

---

## 2. 串口参数

| 参数 | 值 |
|------|------|
| 波特率 | 默认 460800，可配置（见4.2节 `0x03` 命令） |
| 数据位 | 8 |
| 停止位 | 1 |
| 校验位 | 无 |
| 流控 | 无 |

---

## 3. 帧格式

### 3.1 控制帧

用于PC下发指令、STM32返回响应，低频、可靠性优先。

```
+--------+--------+----------+----------+----------+--------+----------+
| 0xAA   | 0x55   | 帧序号   | 命令码   | 数据长度 | 数据   | CRC16    |
| 1 字节 | 1 字节 | 1 字节   | 1 字节   | 1 字节   | n 字节 | 2 字节   |
+--------+--------+----------+----------+----------+--------+----------+
```

| 字段 | 长度 | 说明 |
|------|------|------|
| 帧头 | 2 字节 | 固定为 `0xAA 0x55` |
| 帧序号 | 1 字节 | 发送方递增计数，0~255 循环，响应帧回传相同序号 |
| 命令码 | 1 字节 | 标识指令类型，见第4节 |
| 数据长度 | 1 字节 | 数据字段的字节数，范围 0~255 |
| 数据 | n 字节 | 指令参数或返回内容，由命令码决定 |
| CRC16 | 2 字节 | 对帧序号、命令码、数据长度、数据计算，高字节在前 |

> CRC16 校验范围：帧序号 + 命令码 + 数据长度 + 数据（不含帧头）

---

### 3.2 数据流帧

用于STM32向PC高频上报示波器采样数据，实时性优先，开销最小。

```
+--------+------------+--------+--------+----------+
| 0xBB   | 通道掩码   | LEN    | 数据   | XOR校验  |
| 1 字节 | 1 字节     | 1 字节 | n 字节 | 1 字节   |
+--------+------------+--------+--------+----------+
```

| 字段 | 长度 | 说明 |
|------|------|------|
| 帧头 | 1 字节 | 固定为 `0xBB` |
| 通道掩码 | 1 字节 | 每个bit对应一个通道，bit=1表示该通道有效 |
| LEN | 1 字节 | 数据段字节数，= 通道掩码中bit为1的数量 × 2 |
| 数据 | n 字节 | 各通道采样值，按通道编号从低到高排列，每通道2字节（int16，大端，高字节在前） |
| XOR校验 | 1 字节 | 对通道掩码、LEN和数据所有字节逐字节异或 |

> **I2C 读取失败处理**：若某通道在采样时 I2C 读取失败，该通道数据填 `0x0000`，数据流继续上报，不中断采样。上位机若需区分寄存器值为零和读取失败，应结合应用层逻辑判断。
>
> **I2C 总线故障检测**：固件维护 I2C 连续失败计数器，每个采样 tick 中若所有启用通道的 I2C 读取均失败则计数器递增，任意通道读取成功则计数器归零。连续 10 次全通道失败时，固件发送 `0x06` 调试信息 `"I2C bus stuck, sampling degraded"`。连续 50 次全通道失败时，固件自动停止采样（等同收到 `0x51`）并发送错误响应帧（错误码 `0x03`）及 `0x06` 调试信息 `"I2C bus stuck, sampling auto-stopped"`。

**通道掩码示例：**

| 掩码值 | 含义 |
|--------|------|
| `0x01` | 仅通道0 |
| `0x03` | 通道0、通道1 |
| `0x0F` | 通道0~3 |

---

### 3.3 帧类型识别

接收端通过首字节判断帧类型：

| 首字节 | 帧类型 |
|--------|--------|
| `0xAA` | 控制帧（需继续读取第二字节确认 `0x55`） |
| `0xBB` | 数据流帧 |

## 4. 命令码定义

### 4.1 分组规划

| 范围 | 类别 |
|------|------|
| `0x00` | 心跳 |
| `0x01 ~ 0x1F` | 系统控制类 |
| `0x20 ~ 0x4F` | 寄存器读写类 |
| `0x50 ~ 0x7F` | 示波器与发生器控制类 |
| `0x80 ~ 0xBF` | 预留扩展 |
| `0xC0 ~ 0xFE` | 用户自定义 |
| `0xFF` | 保留 |

---

### 4.2 系统控制组（0x00~0x1F）

#### `0x00` 心跳

| 项目 | 内容 |
|------|------|
| 方向 | 双向 |
| 数据段 | 空（LEN=0） |
| 响应 | STM32 echo 原帧 |

---

#### `0x01` 错误响应

| 项目 | 内容 |
|------|------|
| 方向 | STM32→PC |
| 数据段 | 1 字节错误码（见4.5节） |
| 说明 | STM32遇到异常时主动回复 |

---

#### `0x02` 设置电机 IC 地址

| 项目 | 内容 |
|------|------|
| 方向 | PC→STM32 |
| 数据段 | 1 字节，7-bit I2C 地址 |
| 成功响应 | 同 CMD+SEQ，LEN=0（地址已更新） |
| 失败条件 | 地址为 `0x00` 或 > `0x7F`；I2C 无应答 |
| 失败响应 | 错误响应帧，错误码 `0x03` |
| 备注 | 失败时 `g_motorIcAddr` 保持原值不变 |

---

#### `0x03` 设置波特率

| 项目 | 内容 |
|------|------|
| 方向 | PC→STM32 |
| 数据段 | 1 字节，波特率索引（0x00~0x07） |
| 成功响应 | 用**旧波特率**发送同 CMD+SEQ、LEN=0 的响应帧，发送完成后切换到新波特率 |
| 失败条件 | 索引 > `0x07` |
| 失败响应 | 错误响应帧，错误码 `0x03` |

波特率索引表：

| 索引 | 波特率 |
|------|--------|
| `0x00` | 9600 |
| `0x01` | 19200 |
| `0x02` | 38400 |
| `0x03` | 57600 |
| `0x04` | 115200 |
| `0x05` | 230400 |
| `0x06` | 460800（默认） |
| `0x07` | 921600 |

---

#### `0x04` 系统复位

| 项目 | 内容 |
|------|------|
| 方向 | PC→STM32 |
| 数据段 | 空（LEN=0） |
| 成功响应 | 同 CMD+SEQ、LEN=0 的响应帧，发送完成后执行软件复位 |
| 复位后状态 | 波特率恢复默认（460800），`g_motorIcAddr` 清零，发生器停止，采样停止 |
| 约定 | **上位机发复位前必须先发 `0x03` 将波特率切回 460800** |

---

#### `0x05` 电机应答测试

| 项目 | 内容 |
|------|------|
| 方向 | PC→STM32 |
| 数据段 | 空（LEN=0） |
| 成功响应 | 同 CMD+SEQ，LEN=0 |
| 失败条件 | `g_motorIcAddr` 为 `0x00`；I2C 无应答 |
| 失败响应 | 错误响应帧，错误码 `0x03` |

---

#### `0x06` 调试信息

| 项目 | 内容 |
|------|------|
| 方向 | STM32→PC |
| 数据段 | ASCII 字符串，最长 255 字节 |
| 触发时机 | STM32 主动发送，合法场景见下表 |

固件主动发送 `0x06` 的合法场景：

| 场景 | 示例消息 |
|------|---------|
| 启动采样/发生器失败原因说明 | `"No valid channel mapping"` |
| I2C 总线连续故障告警（连续 10 次全通道失败） | `"I2C bus stuck, sampling degraded"` |
| I2C 总线故障自动停采样（连续 50 次全通道失败） | `"I2C bus stuck, sampling auto-stopped"` |
| I2C 总线恢复通知（RecoverBus 成功） | `"I2C bus recovered"` |
| 启动时耗时超限说明 | `"Tick overrun: estimated Xus > limit Yus"` |
| RX 缓冲区溢出报告 | `"UART RX overflow: N bytes lost"` |

---

#### `0x07` I2C 总线扫描

| 项目 | 内容 |
|------|------|
| 方向 | PC→STM32 |
| 数据段 | 1 字节，总线索引（`0x01`=I2C1 / `0x02`=I2C2 / `0x03`=I2C3） |
| 成功响应 | 数据段：`[count][addr0][addr1]...[addrN-1]`，count 为发现的设备数量，addr 为 7-bit 设备地址 |
| 失败条件 | 总线索引不在 1~3 范围内；总线扫描操作本身失败 |
| 失败响应 | 错误响应帧，错误码 `0x03` |
| 备注 | 扫描地址范围 0x01~0x7E，逐地址发送 START+地址+STOP 检测 ACK |

---

#### `0x08` PMIC LDO 使能

| 项目 | 内容 |
|------|------|
| 方向 | PC→STM32 |
| 数据段 | 空（LEN=0） |
| 成功响应 | 同 CMD+SEQ，LEN=0 |
| 失败条件 | I2C 通讯失败；回读验证 LDO 使能位未置位 |
| 失败响应 | 错误响应帧，错误码 `0x03` |
| 执行序列 | 根据 `g_pmicVoltage` 预设电压，然后按固定顺序使能：LDO2(DRVVDD) → 等10ms → LDO1(IOVDD) → 等10ms → LDO3(VCMVDD)，完成后回读验证 |
| 阻塞行为 | 执行期间（约30ms）不响应其他控制帧，UART RX 中断照常接收存入环形缓冲区 |

---

#### `0x09` 设置 PMIC 电压

| 项目 | 内容 |
|------|------|
| 方向 | PC→STM32 |
| 数据段 | 6 字节：`[DRVVDD_H][DRVVDD_L][IOVDD_H][IOVDD_L][VCMVDD_H][VCMVDD_L]`，大端，值为实际电压(V) × 100 |
| 成功响应 | 同 CMD+SEQ，LEN=0 |
| 失败条件 | LEN ≠ 6；任一电压值超出 60~377 范围（对应 0.60V~3.77V） |
| 失败响应 | 错误响应帧，错误码 `0x03` |
| 备注 | 仅更新系统变量 `g_pmicVoltage`，不立即写入 PMIC 硬件；下次执行 `0x08` 使能时生效。不发此命令则使用默认值 |

电压对应关系：

| 成员 | LDO 通道 | 默认值 | 含义 |
|------|----------|--------|------|
| DRVVDD | LDO2 | 180（1.80V） | 驱动电源 |
| IOVDD | LDO1 | 280（2.80V） | IO 电源 |
| VCMVDD | LDO3 | 320（3.20V） | 共模电源 |

示例：设置 DRVVDD=1.80V, IOVDD=3.30V, VCMVDD=2.50V → 数据段：`00 B4 01 4A 00 FA`

---

#### `0x0A` PMIC LDO 禁用

| 项目 | 内容 |
|------|------|
| 方向 | PC→STM32 |
| 数据段 | 空（LEN=0） |
| 成功响应 | 同 CMD+SEQ，LEN=0 |
| 失败条件 | I2C 通讯失败；回读验证 LDO 使能位未清零 |
| 失败响应 | 错误响应帧，错误码 `0x03` |
| 执行序列 | 按使能的反序关闭：LDO3(VCMVDD) → 等10ms → LDO1(IOVDD) → 等10ms → LDO2(DRVVDD)，完成后回读验证 |
| 阻塞行为 | 执行期间（约20ms）不响应其他控制帧 |

---

### 4.3 寄存器读写组（0x20~0x4F）

#### `0x20` 读单个寄存器

| 项目 | 内容 |
|------|------|
| 方向 | PC→STM32 |
| 数据段 | 2 字节：`[寄存器地址高字节][寄存器地址低字节]`，大端 |
| 成功响应 | 返回帧，数据段 2 字节，大端（高字节在前） |
| 失败响应 | 错误响应帧，错误码 `0x03` |

---

#### `0x21` 写单个寄存器

| 项目 | 内容 |
|------|------|
| 方向 | PC→STM32 |
| 数据段 | 4 字节：`[寄存器地址高字节][寄存器地址低字节][数据高字节][数据低字节]`，大端 |
| 成功响应 | 同 CMD+SEQ，LEN=0 |
| 失败响应 | 错误响应帧，错误码 `0x03` |

---

#### `0x22` 批量读寄存器

| 项目 | 内容 |
|------|------|
| 方向 | PC→STM32 |
| 数据段 | 4 字节：`[起始寄存器地址高字节][起始寄存器地址低字节][读取数量高字节][读取数量低字节]`，大端；读取数量范围 1~32130 |
| 成功响应 | 多帧返回，每帧数据段：`[包序号][总包数][数据...]`，每包最多 252 字节（126个寄存器），每个寄存器 2 字节大端 |
| 失败响应 | 错误响应帧，错误码 `0x03` |
| 备注 | 分包发送过程中，固件在每包发送完成后检查 RX 缓冲区，若有心跳帧（`0x00`）则立即响应，除心跳外的其他控制帧仍忽略；采样进行中收到此命令直接返回错误码 `0x03` |

**地址步长**：起始寄存器地址为字节地址，每个后续寄存器地址递增 **+2**。例如起始地址 `0x0010`，读取 3 个寄存器，依次访问 `0x0010`、`0x0012`、`0x0014`。

**包序号**：从 **0** 开始计数，第一包序号为 `0x00`，最后一包序号为 `总包数 - 1`。

### 4.4 示波器控制组（0x50~0x7F）

#### 采样默认值

| 参数 | 默认值 |
|------|--------|
| 采样间隔 | 索引 `0x03`（400 us） |
| 通道掩码 | `0x01`（仅通道0） |
| 通道寄存器映射 | 全部 `0xFFFF`（未映射） |

---

#### `0x50` 启动采样

| 项目 | 内容 |
|------|------|
| 方向 | PC→STM32 |
| 数据段 | 空（LEN=0） |
| 成功响应 | 同 CMD+SEQ，LEN=0；之后持续上报数据流帧 |
| 失败条件 | 所有通道寄存器映射均为 `0xFFFF`；或启动时预期单 tick I2C 总耗时超过当前 tick 周期的 80%（见「耗时校验」） |
| 失败响应 | 错误响应帧，错误码 `0x03`；同时发 `0x06` 调试信息说明原因 |
| 耗时校验 | 启动时计算预期耗时：`(当前发生器写通道数 × 54μs + 采样启用通道数 × 77μs)`；若预期耗时 > 当前 tick 周期 × 80%，拒绝启动并通过 `0x06` 告知 `"Tick overrun: estimated Xus > limit Yus"` |

---

#### `0x51` 停止采样

| 项目 | 内容 |
|------|------|
| 方向 | PC→STM32 |
| 数据段 | 空（LEN=0） |
| 成功响应 | 同 CMD+SEQ，LEN=0 |
| 说明 | 立即停止 TIM6，停止数据流上报；**通道配置（掩码、寄存器映射）保持不变**，下次 `0x50` 可直接重启采样 |

---

#### `0x52` 设置采样间隔

| 项目 | 内容 |
|------|------|
| 方向 | PC→STM32 |
| 数据段 | 1 字节，采样间隔索引（0x00~0x06） |
| 成功响应 | 同 CMD+SEQ，LEN=0 |
| 失败条件 | 索引 > `0x06` |
| 失败响应 | 错误响应帧，错误码 `0x03` |

采样间隔索引表：

| 索引 | 采样间隔 | 对应频率 |
|------|----------|----------|
| `0x00` | 150 us | 6667 Hz |
| `0x01` | 250 us | 4000 Hz |
| `0x02` | 300 us | 3333 Hz |
| `0x03` | 400 us | 2500 Hz（默认） |
| `0x04` | 500 us | 2000 Hz |
| `0x05` | 900 us | 1111 Hz |
| `0x06` | 1000 us | 1000 Hz |

---

#### `0x53` 设置采样通道

| 项目 | 内容 |
|------|------|
| 方向 | PC→STM32 |
| 数据段 | 1 字节，通道掩码（bit0~bit7 对应通道 0~7） |
| 成功响应 | 同 CMD+SEQ，LEN=0 |
| 失败条件 | 掩码为 `0x00` |
| 失败响应 | 错误响应帧，错误码 `0x03` |

---

#### `0x54` 设置通道寄存器映射

| 项目 | 内容 |
|------|------|
| 方向 | PC→STM32 |
| 数据段 | 16 字节，每通道 2 字节寄存器地址，大端，共 8 个通道按顺序排列 |
| 成功响应 | 同 CMD+SEQ，LEN=0 |
| 不使用的通道 | 填 `0xFFFF` |

---

#### `0x55` 启动线性发生器

| 项目 | 内容 |
|------|------|
| 方向 | PC→STM32 |
| 数据段 | 10 字节（见下表） |
| 成功响应 | 同 CMD+SEQ，LEN=0；STM32 开始本地执行线性递增递减写入 |
| 失败条件 | LEN ≠ 10；min ≥ max；step ≤ 0；intervalMs == 0；或启动后预期单 tick I2C 总耗时超过当前 tick 周期的 80%（耗时校验规则同 `0x50`） |
| 失败响应 | 错误响应帧，错误码 `0x03`；耗时超限时同时发 `0x06` 调试信息 |
| 与采样关系 | **不互斥**，发生器运行期间采样可同时进行 |
| 重复启动 | 收到新的 `0x55` 或 `0x56` 时，自动停止当前发生器再启动新的 |

数据段格式（全部大端）：

| 偏移 | 长度 | 类型 | 说明 |
|------|------|------|------|
| 0 | 2 | uint16 | 目标寄存器地址 |
| 2 | 2 | int16 | 最小值（min） |
| 4 | 2 | int16 | 最大值（max） |
| 6 | 2 | int16 | 步进值（step），必须 > 0 |
| 8 | 2 | uint16 | 写入间隔（ms），近似值，固件自动取最接近采样周期整数倍的值 |

**执行逻辑**：从 min 开始，每隔 interval 毫秒向目标寄存器写入当前值，每次递增 step；若当前值 + step 超过 max，则直接写入 max 再反向递减；递减时同理，若当前值 - step 低于 min，则直接写入 min 再反向递增。到达边界时先写入边界值，下一 tick 再反向。

> **与手动寄存器写入的关系**：发生器运行期间，`0x21` 写同一寄存器地址仍正常执行，但写入值会在下一个发生器 tick 被覆盖。协议层不做冲突保护，由用户自行判断。

> **固件实现约束**：发生器写入与采样读取共享同一条 I2C 总线，必须通过单定时器统一调度，在同一个 tick 内顺序执行（先写后读），避免总线竞争。详见本节末尾「I2C 统一调度」说明。

---

#### `0x56` 启动余弦发生器

| 项目 | 内容 |
|------|------|
| 方向 | PC→STM32 |
| 数据段 | 7 + N×4 字节，N 为通道数（见下表） |
| 成功响应 | 同 CMD+SEQ，LEN=0；STM32 开始本地执行余弦波写入 |
| 失败条件 | LEN < 11（至少 1 个通道）；amplitude ≤ 0；frequency ≤ 0；通道数为 0 或 > 3；或启动后预期单 tick I2C 总耗时超过当前 tick 周期的 80%（耗时校验规则同 `0x50`） |
| 失败响应 | 错误响应帧，错误码 `0x03`；耗时超限时同时发 `0x06` 调试信息 |
| 与采样关系 | **不互斥**，发生器运行期间采样可同时进行 |
| 重复启动 | 收到新的 `0x55` 或 `0x56` 时，自动停止当前发生器再启动新的 |

数据段格式（全部大端）：

| 偏移 | 长度 | 类型 | 说明 |
|------|------|------|------|
| 0 | 2 | int16 | 振幅（amplitude），必须 > 0 |
| 2 | 2 | int16 | 偏移（offset） |
| 4 | 2 | uint16 | 频率 × 100（如 1.50Hz → 150），精度 0.01Hz |
| 6 | 1 | uint8 | 通道数 N（1~3） |
| 7 + i×4 | 2 | uint16 | 第 i 个通道的目标寄存器地址 |
| 9 + i×4 | 2 | int16 | 第 i 个通道的相位 × 10（如 120.0° → 1200），精度 0.1° |

**执行逻辑**：STM32 在统一调度定时器的每个 tick 中，执行余弦写入（写入频率与采样频率一致，无分频）。每次执行时计算各通道当前值并���入对应寄存器：

```
value = offset + amplitude × cos(2π × freq × t + phaseDeg × π / 180)
```

写入值 clamp 到 int16 范围（-32768 ~ 32767）。t 从发生器启动时刻开始算零点。实际写入频率由基础 tick 周期和 I2C 总线带宽决定，不硬编码固定频率。

> **与手动寄存器写入的关系**：与线性发生器相同，发生器运行期间 `0x21` 写同一寄存器地址仍正常执行，但会被下一个 tick 覆盖。协议层不做冲突保护，由用户自行判断。

> **固件实现约束**：与线性发生器相同，余弦写入与采样��取共享 I2C 总线，必须通过单定时器统一调度。详见本节末尾「I2C 统一调度」说明。

#### `0x57` 停止发生器

| 项目 | 内容 |
|------|------|
| 方向 | PC→STM32 |
| 数据段 | 空（LEN=0） |
| 成功响应 | 同 CMD+SEQ，LEN=0 |
| 说明 | 停止当前正在运行的线性或余弦发生器；若无发生器在运行，仍返回成功 |

#### I2C 统一调度（固件实现约束）

发生器（`0x55`/`0x56`）与采样（`0x50`）可同时运行，但共享同一条 I2C 总线。固件必须采用**单定时器统一调度**方案：

- 使用一个硬件定时器作为基础 tick，所有 I2C 操作在同一个 tick 回调内顺序执行
- 每个 tick 内执行顺序：发生器写入（线性发生器通过分频计数器控制实际写入频率，余弦发生器每 tick 都写入）→ 采样读取（每 tick 都执行，TX 忙时跳过）→ 数据流帧打包上报
- 基础 tick 周期必须大于单个 tick 内所有 I2C 操作的总耗时，否则会溢出

软件模拟 I2C（GPIO bit-bang，实测 ~870 kHz）各操作典型耗时参考：

| 操作 | 耗时 |
|------|------|
| 写 1 个寄存器 | ~54μs |
| 读 1 个寄存器 | ~77μs |

典型场景下的最小安全 tick 周期（采样间隔最小 150μs）：

| 场景 | I2C 总耗时 | 建议最小 tick |
|------|-----------|-------------|
| 仅采样 1CH | ~77μs | 150μs |
| 线性 1 写 + 1CH 采样 / 仅采样 2CH | ~131~154μs | 250μs |
| 写 1 + 采样 2CH | ~208μs | 300μs |
| 写 1 + 采样 3CH | ~285μs | 400μs |
| 写 1 + 采样 4CH | ~362μs | 500μs |
| 写 1 + 采样 8CH | ~670μs | 900μs |

> 以上耗时基于软件模拟 I2C（GPIO bit-bang，实测 bit 周期 ~1.15μs，等效 ~870 kHz）、16-bit 寄存器地址、16-bit 数据估算。

#### 采样时间间隔分析

采样周期受三个约束共同决定，取最大值：

```
T_sample ≥ max( T_i2c + T_main,  T_uart,  T_i2c / k )
```

**变量定义：**

| 符号 | 含义 | 值 |
|------|------|----|
| N_r | 采样读通道数（0~8） | 运行时配置 |
| N_w | 发生器写通道数（0 或 1） | 运行时配置 |
| B | 波特率（bps） | 默认 460800 |
| T_read | 单次 I2C 读耗时 | ~70 μs |
| T_write | 单次 I2C 写耗时 | ~48 μs |
| T_overhead | ISR 中 mask 计算、循环、缓冲区切换等 CPU 开销 | ~20 μs |
| T_main | 主循环一轮（Protocol_Poll + Sample_Poll 组帧 + 启动 DMA） | ~50 μs |
| k | CPU 占用率上限（留余量响应控制帧） | 0.9 |

**三条约束：**

| 约束 | 公式 | 含义 |
|------|------|------|
| 约束 1：ISR 不溢出 | `T_i2c + T_main`，其中 `T_i2c = N_w×48 + N_r×70 + 20` | ISR 完成后主循环必须有时间运行 |
| 约束 2：UART DMA 发完 | `(4 + 2×N_r) × 10⁷ / B` | DMA 释放缓冲区后才能发下一帧 |
| 约束 3：CPU 不满载 | `T_i2c / 0.9` | 留 10% 余量处理控制帧 |

> 约束 2 中帧字节数 = 0xBB(1) + mask(1) + LEN(1) + N_r×2 + XOR(1) = 4 + 2×N_r；每字节 10 bits（UART 8N1）。

**460800 bps 速查表：**

| N_r | N_w | 约束1 (μs) | 约束2 (μs) | 约束3 (μs) | **T_sample (μs)** | **瓶颈** | **最大频率** |
|-----|-----|-----------|-----------|-----------|-----------------|--------|------------|
| 1 | 0 | 140 | 130 | 100 | **140** | I2C+主循环 | 7143 Hz |
| 1 | 1 | 188 | 130 | 153 | **188** | I2C+主循环 | 5319 Hz |
| 2 | 1 | 258 | 174 | 231 | **258** | I2C+主循环 | 3876 Hz |
| 4 | 1 | 398 | 260 | 387 | **398** | I2C+主循环 | 2513 Hz |
| 8 | 0 | 630 | 434 | 689 | **689** | CPU 占用率 | 1451 Hz |
| 8 | 1 | 678 | 434 | 742 | **742** | CPU 占用率 | 1348 Hz |

> 460800 下 UART（约束 2）在所有配置中均不是瓶颈，进一步提高波特率无收益。瓶颈为 I2C 读写耗时和 CPU 占用率。

**推荐档位映射（460800 bps）：**

| 通道配置 | T_sample | 推荐最快档位 |
|---------|---------|------------|
| 1 读 + 0 写 | ≥ 140 μs | idx 0（150 μs / 6667 Hz） |
| 1 读 + 1 写 | ≥ 188 μs | idx 1（250 μs / 4000 Hz） |
| 2 读 + 1 写 | ≥ 258 μs | idx 2（300 μs / 3333 Hz） |
| 4 读 + 1 写 | ≥ 398 μs | idx 3（400 μs / 2500 Hz） |
| 8 读 + 0 写 | ≥ 689 μs | idx 5（900 μs / 1111 Hz） |
| 8 读 + 1 写 | ≥ 742 μs | idx 5（900 μs / 1111 Hz） |

**115200 bps 参考（旧配置）：**

115200 下所有通道配置均由 UART（约束 2）主导。8 通道时 T_uart = 1736 μs，实际有效上限仅 ~576 Hz。

### 4.5 错误码定义

| 错误码 | 名称 | 说明 |
|--------|------|------|
| `0x01` | CRC校验失败 | 收到的控制帧CRC校验不通过，错误响应帧序号固定填 `0xFF` |
| `0x02` | 未知命令码 | 命令码未定义或不支持 |
| `0x03` | 执行失败 | 命令执行过程中发生错误（如I2C通讯失败） |

## 5. 心跳机制

- PC端每隔 **1秒** 发送一次心跳控制帧（命令码 `0x00`，数据长度 `0x00`）
- STM32收到后立即回复相同格式的心跳帧
- PC端若连续 **3次** 未收到心跳回复，判定连接断开，界面给出断线提示

---

## 6. 超时与重发机制

- 本机制仅适用于普通控制帧，**心跳帧独立处理，不走本机制**
- PC发出控制帧后，等待STM32响应超时时间为 **100ms**
- 超时后自动重发，最多重发 **2次**（共发送3次）
- 3次均无响应则判定本次指令失败，上层报错
- 重发时帧序号保持不变，STM32收到重复帧序号时重新执行并回复（不做去重处理）

---

## 7. CRC16 计算说明

- 算法：CRC16（多项式 `0x8005`，初始值 `0xFFFF`，右移、低位先行）
- 注意：本协议直接使用 `0x8005` 配合右移算法，与标准 CRC16-MODBUS（右移 + 反射多项式 `0xA001`）不同，不可混用标准 Modbus CRC 库
- 校验范围：帧序号 + 命令码 + 数据长度 + 数据
- 结果2字节，**高字节在前（大端）** 存入帧
